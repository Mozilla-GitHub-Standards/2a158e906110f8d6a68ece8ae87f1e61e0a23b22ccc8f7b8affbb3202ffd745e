Module Updates
LDAP
August 20
Submitted by Dan Mosedale <dmose@netscape.com>
  • Spent much of the week shaking out issues with my patch for bug 89198 (fixes RFC 822 quoting for LDAP autocomplete display names). Checked in the changes on Friday. The changes for this also cleaned up some of the error-handling code for when LDAP setup in the compose window fails for one reason or another. If there were any users of the LDAP autocomplete "outputFormat" hidden preference, that has now been split up into three separate hidden prefs: "addressFormat", "nameFormat", and "commentFormat". See the bug for details.
  • Iterated through a couple of versions of a patch to the LDAP C SDK to make aMozilla not stall out during a failed connect (bug 79509). Still need at least one more iteration. Thanks to Mark Smith (mcs@netscape.com) for help sorting out the issues here.
  • Helped Leif shake out design issues with async DNS changes for the LDAP XPCOM SDK (bug 82142).
  • Reviewed and built a patch (bug 17879) from John Marmion (john.marmion@ireland.sun.com) to address other review issues with LDAP addressbook code. Awaiting sr=.
