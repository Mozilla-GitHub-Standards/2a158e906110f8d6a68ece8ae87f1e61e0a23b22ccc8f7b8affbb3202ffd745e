Language Packs
Mar 3
Submitted by Tao Cheng <tao@netscape.com>
Thanks to Dawn's assistance, I had moved all submitted language packs, localized builds, and localization tools to http://ftp.mozilla.org/pub/mozilla/l10n/. This shall take down the size of files on Mozilla's documents server and also improve the visibility of our localization contribution. The directory structure looks like this: 

        http://ftp.mozilla.org/pub/mozilla/l10n/  --> top directory

                glossaries/     --> contains all 4.x glossary files
                        *.txt *.utf8

                lang/   --> contains all localized builds/lang-packs
                        m8/ ... m14/  
                            mozilla-[os][-lang|-talkback]-[locale].{zip, tar.gz}

                tools/  --> contains all localization/leveraging tools
                        mozexptool/
                        mozillatransaltor/
All future submission will be pushed to the appropriated directories. Feel free to let me know if you have access problem.

    Crypto
    Mar 6
    Submitted by Bob Lord <lord@netscape.com>
    We recently landed some of the crypto code onto the M14 branch. In particular, we landed stuff like the HTTPS protocol hander, and the PSM client libraries (see the PKI page for more info on PSM: http://www.mozilla.org/projects/security/pki/). This landing will make it possible for people to use PSM with Mozilla daily builds in addition to the Milestone builds.

    We're not landing PSM source just yet, though we are working our way toward that goal. See the FAQ for more details: (http://www.mozilla.org/crypto-faq.html#1-2). Also, we are trying to build an XPI file so that you can quickly add the PSM binary to Mozilla with one or two mouse clicks. As soon as it works at all, we'll publish a page with instructions and ask for help testing.

    March 12:

    This past week we released a version of PSM which works with Mozilla's M14 branch. Now you can shop with Mozilla! See this page for more info.

    After Netscape branched for Beta, Pav migrated all our changes to the trunk.  PSM should work with our nightly builds starting in the next day or so.
