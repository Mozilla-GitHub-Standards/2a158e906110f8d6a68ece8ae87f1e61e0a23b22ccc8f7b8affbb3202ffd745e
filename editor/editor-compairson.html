Gecko Editor vs. MSHTML (IE) Editor
10/25/2002

Purpose of this document

The purpose of this document is to compare the current state of Gecko embedded HTML editing to MSHTML (IE) HTML editing.

10 Second Summary

The Gecko embeddable HTML editor is comparable to the MSHTML embeddable editor in basic functionality. The divergence is around more advanced features and embedding the editor in web content versus in an application.  

There is also some divergence in core editor behaviors, and content produced.

Functional Divergence

By their very nature, Gecko and IE are different, so the markup each editor can produce and accept does diverge in some cases. There are white space handling differences. There are UI differences. Some of these things are considered bugs, some are not. To get a feeling for the types of issues, you can query the "core" editor bug list with this Bug List.

The highest priority issues from that list for getting fixed are in this list. This may overlap with, but is not totally inclusive of issues that are also being working on for specific projects at any given time.
