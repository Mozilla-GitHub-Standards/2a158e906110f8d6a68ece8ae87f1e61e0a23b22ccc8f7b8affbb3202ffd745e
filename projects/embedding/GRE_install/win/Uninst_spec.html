GRE Uninstaller Specifications
(Windows Platforms)

The GRE Installer for Windows will install an uninstaller program which will provide the following functionality:
  • Gather User input:
    • Relevant command-line options
      • "/ua" <user_agent> - Defines the version and language  (no change)
      • "/app <app_id>"    (new)
        • Forces removal of the entry from the AppList for this app for the path designated in app_path below.
        • app_id must be "GreUser" for user uninstall.
      • "/app_path <path_to_exe>"    (new)
        • Must be provided if the app can be installed in multiple locations.  The AppList will be updated only for the app installation in this location.
        • If not provided all the app will be removed from the AppList regardless of how many installations exist on the system.
      • "/reg_path <path_to_alternative_registry_location>"  (new)
        • private copies must use this to identify where they are registered in the Windows  registry
      • "/reg_path <path_to_alternative_registry_location>"  (new)
        • private copies must use this to identify where they are registered in the Windows  registry
      • "/v" - Verbose: shows info about what apps are using GRE if uninstaller decides not to uninstall   (new)
      • "/ms" - There is currently UI that is going to have to be turned off.
  • Determine whether to physically Uninstall (and clean up orphaned entries in AppList while we're at it): 
    • Determine the root path in the registry (we'll refer to this as GREROOT)
      • If the command-line option /reg_path has been set, use that
      • Else GREROOT is "HKLM\Software\Mozilla\GRE"
    • For each app_id listed under "GREROOT\<version_number>\AppList\"
      • If the app_id matches the app_id from the the command-line (i.e. /app)  //We want to remove GRE for this app
        • If  (the app_id is GreUser) Or (no app_path was given on the command-line)
          • delete "GREROOT\<version_number>\AppList\<app_id>" and all keys under it
          • If the app_id is GreUser
            • Remove GRE entry from Windows registry Add/Remove Programs list
        • Else  //Since we got a app_path from the user, make sure we only remove GRE for that installation of the app
          • For each PathToExeXX entry
            • (If the value matches the app_path from the command-line) Or (the file pointed to by PathToExeXX does not exist)
              • delete the PathToExeXX key
              • re-enumerate the PathToExeXX keys
            • else
              • increment the AppCount
          • If no PathToExe entries remain for app_id
            • delete "GREROOT\<version_number>\AppList\<app_id>" and all keys under it
      • Else If the app_id is GreUser  //Special case since it does not have any PathToExe entry
        • If there is an unistaller entry in the Windows registry Add/Remove Programs list
          • increment the AppCount
        • Else
          • delete "GREROOT\<version_number>\AppList\<app_id>" and all keys under it
      • Else 
        • For each PathToExeXX entry
          • If the file identified by PathToExe actually exists on the machine
            • increment the AppCount
          • Else
            • delete the PathToExeXX key
            • re-enumerate the PathToExeXX keys
      • If no PathToExe entries remain for app_id
        • delete "GREROOT\<version_number>\AppList\<app_id>" and all keys under it
    • If GREUser is was not in the AppList, remove Gre from the Add/Remove Programs list also.
    • If  AppCount equals 0
      • Uninstall = TRUE
      • delete "GREROOT\<version_number>" and all keys under it
  • If Uninstall is TRUE, read through logfile:
    • Perform uninstall of all files and registry settings based on the list in the uninstaller
  • If Uninstall is FALSE ..And.. /v command-line option
    • Display dialog showing what apps are still installed on the machine for this version of GRE.
  • Return error value
    • 0 means success
    • 1 means failure
