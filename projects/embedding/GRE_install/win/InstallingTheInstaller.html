Installing the Stub Installer

Purpose:  Since we are requiring that app installers launch the GRE installer to perform all the administrative details associated with a shared app, we want to provide an option to the application installers to use a GRE installer from the disk rather than downloading the installer.  The application installer can look in the Windows registry to determine if the GRE installer is available from disk.  If it is not, the app installer knows that it must download the full GRE blob installer.  If it is, GRE is assumed to have already been installed, and the GRE stub installer can be launched from disk.  Following are some considerations which may require further discussion:
  • Absence of the installer implies that GRE has not been installed.  It is possible that user might have deleted the uninstaller even though GRE is really installed.  In this case the full blob installer would be downloaded unnecassarily, but without any other negative consequences.  Since I intend to install the installer with the GRE, I don't this is very likely to happen.  I would expect that the user would remove the entire GRE (bad user!) if he/she were going to do anything.  Even so, if this scenerio does occur, the consequences are minimal.
  • This is a little more interesting:  What if the user removes GRE without removing the installer?  The application, finding the installer on the disk, will assume GRE is installed and will not download the full blob installer.  Since my intention is to install only the stub installer, not the full blob installer, the GRE stub installer will notice that GRE is not really installed and will download the GRE bits.  This is something we are trying to avoid.  But it will only happen if the user "misbehaves", and, for the same reasons I stated in the point above, I think this is very unlikely to happen.  If really, really, really, don't want to take a chance of this ever happening, we could install the full blob installer (or its equivalent) instead of the stub.  The downside to this would be wasted disk space.  The upside would be that the bits would always be available without downloading for recovery of a corrupted installation etc.
