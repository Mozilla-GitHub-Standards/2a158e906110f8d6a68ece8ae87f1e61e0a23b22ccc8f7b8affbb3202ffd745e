Test Scenerios:  (Client A and Client B are designated from the command-line.  Batch files could be created to do this.)

  • Test command-line options:  See Installer Spec and Uninstaller Spec
  • Install by Client A, uninstall by Client A:  
    • On install, files get copied to correct location and registry is updated for Client A.  
    • On uninstall all files and registry settings are removed.
  • Install by Client A, install by Client B, uninstall by Client A, unistall by Client B:  
    • On install by Client A, files get copied to correct location and registry is updated for Client A
    • On install by Client B, no files are copied, registry is updated for Client B
    • On uninstall by Client A, no files are removed, registry is updated to remove Client A
    • On uninstall by Client B, all files and registry settings are removed.
  • Install with GUI, uninstall from "Windows Add/Remove Programs" dialog:  
    • Test all dialogs:  text and buttons.
    • On install files get copied to correct location and registry is updated for "GreUser"
    • On uninstall all files and registry settings are removed.
  • Install with GUI, install by Client A, uninstall from "Windows Add/Remove Programs" dialog, uninstall by Client A:  
    • On install with GUI, files get copied to correct location and registry is updated for "GreUser"
    • On install by Client A, no files are copied, registry is updated for Client A
    • On manual uninstall, no files are removed, registry is updated to remove GreUser.
    • On uninstall by Client A, all files and registry settings are updated to remove Client A
  • Install with GUI, install by Client A, uninstall by Client A, uninstall from "Windows Add/Remove Programs" dialog:  
    • On install with GUI, files get copied to correct location and registry is updated for "GreUser"
    • On install by Client A, no files are copied, registry is updated for Client A
    • On uninstall by Client A, no files are removed, registry is updated to remove Client A
    • On manual uninstall, all files and registry settings are updated to remove GreUser
  • Install by Client A, upgrade by Client A, uninstall by Client A:  
    • On install, files get copied to correct location and registry is updated for Client A
    • On upgrade, files get copied to correct location for new version, registry is updated for new version of GRE for Client A, all files and registry settings are removed for old version of GRE.
    • On uninstall all files and registry settings are removed.
  • Install by Client A, install by client B, upgrade by Client A, upgrade by Client B, uninstall by Client A, uninstall by Client B:  
    • On install by Client A, files get copied to correct location and registry is updated for Client A
    • On install by Client B, no files are copied, registry is updated for Client B
    • On upgrade by client A, files get copied to correct location for new version, registry is updated to remove Client A from the AppList for the old version and add Client A to the AppList for the new version.
    • On upgrade by client B, no files are copied for new version of GRE, registry is updated to add Client B to the AppList for the new version, all files and registry settings are removed for the old version of GRE.
    • On uninstall by Client A, no files are removed, registry is updated to remove Client A
    • On uninstall by Client B, all files and registry settings are removed.
  • Install by Client A, install by client B, upgrade by Client A, uninstall by Client B:  
    • On install by Client A, files get copied to correct location and registry is updated for Client A
    • On install by Client B, no files are copied, registry is updated for Client B
    • On upgrade by client A, files get copied to correct location for new version, registry is updated to remove Client A from the AppList for the old version and add Client A to the AppList for the new version.
    • On uninstall by Client B, all files and registry settings for the old version of GRE are removed.
