Mozilla Tech Evangelism History

To understand the reason for evangelism, it is necessary to know about the browser wars. The browser wars occurred around 1996 between Microsoft and Netscape primarily. They were both trying to gain the largest share of the browser market.

Browser War

With the debut of Windows 95, the way users accessed the World Wide Web (W3) changed. Before Windows 95 and Internet Explorer, the mainstream browser was Netscape Navigator, which was a product that cost money at the time. With the release of Windows 95, Internet Explorer was given away to all users virtually for free. This led to competition in which each browser incorporated more proprietary features in their efforts to gain the largest share of users.

Because of the many proprietary features, the jobs of webmasters were quickly made cumbersome. Websites had to be tested not only in the two different browsers, but also on different platforms with different versions of the same browser. Webmasters often used Internet Explorer's marquee element and Navigator's blink element. That was just the start of the trouble.

For interactivity, each company developed its own Document Object Model (DOM) to allow interactivity through the Dynamic Hypertext Markup Language (DHTML). As these companies created their own DOM using proprietary techniques such as Internet Explorer's document.all and Navigator's document.layers, the World Wide Web Consortium (W3C) was developing the free and open DOM to become the standard. This furthered the confusion.

Web developers did not follow the standards because the browsers did not follow the standards and browsers did not follow the standards because web developers did not follow the standards. This was about to change.

mozilla.org and NGLayout

On January 23, 1998, Netscape Communications decided to make a major change. They announced that they would make Netscape Communicator freely available. That was not all though. They stunned the whole computer industry when they announced that the source code would also be open and free.

They created mozilla.org to fill the role of a benevolent dictator to direct the development of the Mozilla source code. It was quickly decided that the rendering engine from Netscape Navigator 4 would need a major rewrite. Instead of fixing the code, mozilla.org decided to create a new layout engine known as the Next Generation Layout Engine (NGLayout). Netscape Communications publicized this rendering engine by giving it the more well-known nickname of "Gecko."

When the new rendering engine was complete, it was found that many existing websites still used the proprietary techniques that did not work correctly with the standards-compliant NGLayout. The mozilla.org community decided to mark these bugs as invalid using the mozilla.org bug reporting system known as Bugzilla.

Something Needed Doing

All of these invalid bugs called for action.

Evangelism Begins

mozilla.org realized that something needed to be done so they created an evangelism component within the Browser product and started sending evangelism bugs there. Oddly, this component was assigned to nobody@mozilla.org with a Quality Assurance (QA) contact of nobody@mozilla.org. Eventually, timeless@mac.com came along and wanted a real person as the Default QA Contact and Default Owner for every component. Zach Lipton became the Default QA Contact and Blake Ross became the component's Default Owner.

Evangelism Evolves

Eventually, Netscape became more committed to the Tech Evangelism effort and hired a team to help with evangelism. Bob Clary stepped up to the challenge and on April 27, 2001, he became the Default Owner for the evangelism component according to bug 77885.

The evangelism component eventually became too large and uncategorized so in August of 2001, the evangelism component became its own product with its own components; this is documented in bug 86997.
