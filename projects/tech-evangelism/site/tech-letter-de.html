Wenn Sie den Empfehlungen der ECMA für ECMAScript (JavaScript) und den W3C-Empfehlungen für HTML 4.01, CSS 2.1 und DOM (Document Object Model) in Ihrem Webangebot entsprechen, wird Ihre Webseite sowohl korrekt mit diesen Browsern der nächsten Generation zusammenarbeiten, als auch mit Browsern wie dem Opera 9, Safari 2 und Internet Explorer 7 welche offene Standards unterstützen.

Die folgenden Ressourcen helfen Ihnen, den Offenen Standard zu unterstützen und Ihre Webseiten mit Microsoft Internet Explorer korrekt anzuzeigen. Diese Informationen beinhalten auch Wichtiges über die Erkennung von Gecko-basierten Browsern.

ECMAScript 262 3rd Edition (JavaScript) Standard
------------------------------------------------
http://www.ecma-international.org/publications/standards/Ecma-262.htm (englisch)

World Wide Web Consortium (W3C) - Technische Empfehlungen
---------------------------------------------------------
Das W3C definiert die öffentlichen Standards für das World Wide Web (WWW), welche Firefox 2.
http://www.w3.org/TR/ (englisch)
http://www.w3.org/2005/11/Translations/Lists/ListLang-de.html (deutsche Übersetzungen)

Mozilla Developer Center Webentwickler-Dokumentationen
------------------------------------------------------
Besuchen Sie die Webentwickler-Seite im Mozilla Developer Center für genauere Informationen zur Unterstützung von Firefox und anderen Browsern, die auf der Gecko-Layout-Engine basieren.
http://developer.mozilla.org/de/docs/Hauptseite

Verwenden von Web-Standards in Ihren Webseiten
----------------------------------------------
Einige Elemente und Methoden vor allem im Bereich DHTML und/oder Cross-Browser-Unterstützung werden unter Webentwicklern als "Standards" angesehen, obwohl Sie den W3C-Spezifikationen für HTML 4.01 und DOM widersprechen. Finden Sie heraus, was unterstützt wird und was nicht.
http://developer.mozilla.org/en/docs/Using_Web_Standards_in_your_Web_Pages (englisch)

Das Document Object Model in Mozilla
------------------------------------
http://www.mozilla.org/docs/dom/ (englisch) beinhaltet eine Vielzahl von Links über das Lernen und Anwenden des Document Object Model (DOM) in Mozilla und anderen auf Gecko basierenden Browsern.

Die Gecko-DOM-Referenzen http://developer.mozilla.org/en/docs/Gecko_DOM_Reference (englisch) unterstützen Webentwickler-Einsteiger mit Anleitungen und Beispielen zu Firefoxs Implementation des W3C DOM.

Aktualisieren von DHTML-Webseiten
---------------------------------
Alles, was Sie über document.all und alle anderen proprietären Erweiterungen wissen sollten und wie man damit in einer Cross-Browser-Welt umgeht.
http://developer.mozilla.org/de/docs/Web-Standards

Browsererkennung und Cross-Browser-Unterstützung
------------------------------------------------
Wenn Sie auf Gecko basierende Browser erkennen wollen, ist es das einfachste, nach dem String "Gecko" zu suchen. Dies ist viel besser als nach einem bestimmten Hersteller oder gar nach einem ganzen User-Agent-Bezeichner zu suchen. Mehr Informationen finden Sie in diesem Artikel:
http://developer.mozilla.org/en/docs/Browser_Detection_and_Cross_Browser_Support (englisch)
http://developer.mozilla.org/en/docs/Using_Web_Standards_in_your_Web_Pages#Developing_cross_browser.2Fcross_platform_pages (englisch)

Venkman ist ein vollwertiger interaktiver Debugger für Firefox und Seamonkey. Für mehr Informationen lesen Sie bitte:
http://developer.mozilla.org/en/docs/Venkman (englisch)
http://www.svendtofte.com/code/learning_venkman/ (englisch)
