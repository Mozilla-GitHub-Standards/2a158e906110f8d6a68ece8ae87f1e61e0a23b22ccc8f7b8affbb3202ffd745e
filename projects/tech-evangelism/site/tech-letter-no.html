Til Webmasteren

Ditt nettsted TILPASS: sett inn URL her støtter ikke Firefox-nettleseren eller andre nettlesere basert på layoutmotoren Gecko, som også er brukt av bl.a. Camino 1, Seamonkey 1, Epiphany 2, Galeon 2 og K-meleon 1.

Det er lagt inn en rapport om problemet i mozilla.orgs feilbehandlingssystem (bug system). Du kan se kommentarer og kommentere selv:
https://bugzilla.mozilla.org/show_bug.cgi?id=TILPASS: sett inn bugnummer her

TILPASS: Legg inn en teknisk beskrivelse av problemene her, sammen med anbefalinger om hvordan problemet skal rettes. Du kan også legge ved fungerende eksempler som viser hvordan nettstedets kode kan endres for å støtte standardene og Mozilla.

Ved å følge ECMA-anbefalingene for EMCAScript (JavaScript) og W3C-anbefalingene for HTML 4.01, CSS 2.1, og DOM (Document Object Model) i innholdet deres, vil nettstedet deres fungere med disse nestegenerasjons-nettleserene, i tillegg til andre nettlesere som støtter åpne standarder som Opera 9, Safari 2 og Internet Explorer 7. Husk også at nettlesere i håndholdte PCer, mobiltelefoner o.l. ikke nødvendigvis viser websider med feil i på samme måte som en bestemt versjon av MS IE7. Dessuten er det mange som ikke vil eller kan kjøre MS IE7 da de bruker andre plattformer enn MS windows, f.eks. Linux som stadig flere bruker. Og www.skolelinux.no er skolene i ferd med å ta i bruk (i den finnes ikke MS IE, men Firefox og Opera).

Følgende ressurser vil hjelpe deg å støtte åpne standarder samtidig som nettstedet ditt fortsatt vil fungere med Microsoft Internet Explorer. Informasjonen om identifisering av Netscape Gecko-nettleseren er også viktig.

ECMAScript 262 3rd Edition (JavaScript) Standard
------------------------------------------------
http://www.ecma-international.org/publications/standards/Ecma-262.htm

Tekniske anbefalinger fra World Wide Web Consortium (W3C)
---------------------------------------------------------
W3C definerer de offisielle standardene for World Wide Web som Firefox bruker.
http://www.w3.org/2005/11/Translations/Lists/ListLang-no.html
http://www.w3.org/TR/

Automatisk sjekk av HTML-sider
------------------------------
W3C har en validatorer for HTML og CSS, som raskt og enkelt vil gi deg oversikt over feilene i dine HTML-sider og informasjon om hvordan disse skal rettes. Man kan både oppgi en URL eller laste opp lokale filer for sjekk.
For HTML: http://validator.w3.org/
For CSS: http://jigsaw.w3.org/css-validator/

Automatisk retting av HTML-sider
--------------------------------
Her er et program som retter opp vanlige feil i dine HTML-sider automatisk, og samtidig gjør sidene mer lesbare, med f.eks. innrykk. Programmet finnes for mange plattformer. Husk likevel å teste sidene mot validatoren nevnt over etter å ha brukt tidy.
http://tidy.sourceforge.net/
http://users.skynet.be/mgueury/mozilla/index.html

Dokumentasjon for webutviklere på Mozilla Developer Center
---------------------------------------------
Besøk mozilla.orgs webutvikler-side for informasjon om hvordan du bedre kan støtte Firefox og andre nettlesere basert på layoutmotoren Gecko.
http://developer.mozilla.org/en/docs/Web_Development

Bruke webstandarder i dine websider
-----------------------------------
Et antall elementer og praksiser betraktet som "standard" blant webutviklere for å legge til DHTML og/eller kryssnettleser-støtte i sine websider ble ekskludert fra W3C HTML 4.01 og Document Object Model-spesifikasjonene. Finn ut hva som er støttet.
http://developer.mozilla.org/en/docs/Using_Web_Standards_in_your_Web_Pages

Gecko DOM-referanse
-------------------
Gecko DOM-referansen gir nybegynnere innen webutvikling en guide til Firefoxs implementasjon av W3C DOM og inkluderer mange eksempler.
http://developer.mozilla.org/en/docs/Gecko_DOM_Reference

Oppdatering av DHTML-websider
-----------------------------
Det du trenger å vite om document.all og andre proprietære tillegg og hva man gjør med disse i en verden med flere ulike nettlesere.
http://developer.mozilla.org/en/docs/Web_Standards

Nettleser-deteksjon og kryssnettleser-støtte
--------------------------------------------
Når du sjekker om leserene av nettstedetet ditt kjører nettlesere basert på layoutmotoren Gecko, er det enkleste for deg å se etter "Gecko" i user-agent-strengen, ikke leverandørnavn eller en hel user-agent-streng. Du vil finne flere detaljer i denne artikkelen:
http://developer.mozilla.org/en/docs/Browser_Detection_and_Cross_Browser_Support
http://developer.mozilla.org/en/docs/Using_Web_Standards_in_your_Web_Pages#Developing_cross_browser.2Fcross_platform_pages

JavaScript-debugger for Firefox, Seamonkey
-------------------------------------------
Venkman er en komplett interaktiv javascript-feilsøkingsprogram (debugger) for Firefox og Seamonkey som kjører på alle plattformer hvor Firefox og Seamonkey er støttet. For en introduksjon, vennligst les:
http://developer.mozilla.org/en/docs/Venkman
http://www.svendtofte.com/code/learning_venkman/
