Multiple Identities UI Spec

Thunderbird (0.5 and higher) and now the Mozilla Suite version 1.7 support the notion of multiple identities for a given e-mail account. Please read: Configuring Multiple Identities to learn more about what this feature means.

Currently a user must manually create these identities by modifying the profile's prefs.js file. We need to have a clean, simple UI that properly exposes this functionality from within the application. The UI needs to:

  • Allow the user to create, edit and modify multiple identities for an account
  • Each identity should have fields for: name, e-mail address, reply-to address, organization, attached vCard, and a field for an attached signature.
  • Optionally, we may expose the notion of a default identity. The default identity is the one that shows up first in the list of identities. It is also the one whose information we show in the default account settings panel. It is unclear if we really need to distinguish this. We may punt on this requirement.

There are a lot of similarities between the desired multiple identities UI and the existing UI for managing multiple SMTP servers from the account manager. We will use the same model developed for multiple SMTP servers for the identies UI.

Account Manager Integration

Add a button labled "Manage Identities..." to the default account settings panel for IMAP, News and POP3 accounts. This button appears inside the "Identity" box. This button should appear at the end of the Identity box after the attach vCard button. Clicking this button launches the new identity list dialog.

Identity List Dialog

A simple dialog which lists the formatted names for all of the identities in this account. To the right of this list box are buttons for the following actions:

  • Add --> to add a new identity to this account
  • Edit --> to edit the selected identity for this account
  • Delete --> to delete the selected identity for this account (Don't allow the user to delete the last identity

We won't support canceling out of this dialog. We should have just an OK button to dismiss the dialog.

Identity Properties Dialog

This dialog is invoked from the Identity List dialog when the user chooses to add or Edit an identity. It needs to expose the following fields:

  • Name
  • E-mail Address
  • Reply-To Address
  • Organization
  • Attach vCard (with an edit vCard button)
  • Attach signature option

The dialog should have the same layout as the Identity group box in the main account settings panel in the account manager, offering the same fields. Clicking the ok button saves the identity and adds it to the underlying identity list dialog. Cancel, cancels out of the operation.

Other Issues

Down the line we may want to consider adding other features to this UI but for now let's keep it simple. We might also add properties for security preferences and the ability to set the identity as the default identity for the account. The default identity is the identity that shows up pre-filled in the Account Settings panel.
