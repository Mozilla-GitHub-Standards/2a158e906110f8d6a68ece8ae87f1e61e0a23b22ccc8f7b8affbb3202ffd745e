UI Specification

Improving Ease of Use for Tabbed Browsing

Last Modification:


Author:
Marlon Bishop


Feature Team:

Peter Annema - Engineering
Marlon Bishop - UE
Patrice Caire - UE
Sarah Liberman - QA

Summary

Based on usability testing, Netscape UE has identified a few potential areas for improving tabbed browsing. This document is a summary of how we sought to improve the problem areas and is now open for feedback. Please comment in relevent bugs, or to: Bugzilla 188583



TB1.0 Move the [X] close box from the right side of tab bar to the focused tab (not every tab)

Users did not immediately recognize this critical widget as well as had hoped. Just under half relied on right click, or menus as workarounds. Placing it here will greatly increase visibility and directly increase ease of use with closing multiple tabs; require less mousing to and fro. Known issues:
  • Bug 117077
  • Bug 108938

TB2.0 Groupmarks

Several ideas for improving groupmarking and behavior:
  1. When launched, groupmarks should append to the current tab (set)
  2. Enable [open in new window] where right clicking is available
  3. Consider putting an "Always Open This Group in New Window" checkbox in a groupmark properties dialog

TB3.0 Tab Visual Improvements

In general, users failed to notice tabs. Once they gained an understanding of the page tab concept they immediately grasped how to use tabs. If we increase the visual prominence (without detracting from UE or overwhelming the toolbar) then we can improve identification of the feature. The new tab button placement seemed to be appropriate, however the icon itself might need to be clearer.
  1. Jag to investigate 8 slice image buttons (instead of css styling which is being used now) - report the impact on performance
  2. Jag to investigate complexity and impact of doing overlapping tab affordance (using images) - report the impact on performance
  3. Marlon to design new tabs and tab bar images
  4. Marlon to design New Tab Icon
  5. Marlon to unify the close box (between sidebar and tab bar)

TB4.0 Getting the tab bar back

Once closed, users could not find an easy way to get the tab bar back; or, it was that the method to reopening is not 1:1 in terms of e. We did not see any user desire to close the tab bar, as with sidebar, due to its consumption of space. Therefore, we can assume that tab bar clutter and space robbing isn't as significant as with sidebar's. Users had no problem with the current behavior of closing tab bar, however, as stated previously, if they did close it, it was not apparent how to get it back. Implementing TB1.0 solves this problem, btw.
  1. Jag to disable the close box when one tab is in view. (dependent on TB1.0)
  2. Get it back/make it go away, with right click on toolbar

TB5.0 Tab Order When Closing

When closing tabs in a multi-tab situation, the next tab to come into view should be option number one from below:
  1. the tab to the left (preceded in space)
  2. the tab to the right (next in space)
  3. the most previously viewed tab (preceded in time)

While option number 3 sounds reasonable, it implies a hidden z-order which would rely too much on user long term memory to be predictable. Since there is no way of representing z-order visually this cannot be an option in our current implementation. In the usability test, users did not seem to be impeded by going either direction (to the left or to the right), as long as it was something they could rely upon and learn. Possibly a more intensive study would reveal which direction is more natural to most users, given a larger variety of real world scenarios.

TB6.0 Closing All Tabs (with the exception of the current one)

In labs and in community, users expressed the desire to close every other tab except the one currently in view. We currently provide this in contextual menus when clicking on the tab or tab bar area -> [Close Other Tabs]. However although the functionality was available, user's didn't quite understand the term for the menu item, or didn't discover it since it was at the bottom of the menu.
  1. Consider Renaming the Menu Item from [Close Other Tabs] to [Close All Tabs], or something else.
  2. Consider moving the menu item so that it appears higher up:

Tab Menu:

New Tab
Close Tab
Close Other Tabs
Reload Tab
Reload All Tabs

or, with a name and position change.

New Tab
Close Tab
Close All Tabs
Reload Tab
Reload All Tabs
