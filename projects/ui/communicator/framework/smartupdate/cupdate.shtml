Design Cheatsheet

Continuous Update Flowchart

  

1

Update becomes available on the net. The client finds out about it or it is informed.

 

2a

User has notification on: Prompt user before beginning download.

2b

User has notification off: Begin download in background according to preference settings.

 

 

3

Download Software

 

4a

If the jar is signed and the signature is verifiable, then begin installation without prompting the user. 

4b

If the jar is not signed, or if this jar file explicitly asks to prompt the user, then prompt the user before installing the package.

 4c

If the user has checked "Require manual confirmation of each install" then prompt the user before beginning the installation.

 

5

Notify the user that the new software is available for use.

 

Design Details

Visually, the dialogs presented to the user should be similar in look and feel to the Smart Update dialogs. We need to present a single face for software updates.

Dialogs should not intimidate the user. JavaScript security dialogs are currently out of the scope of this specification. (Let me know if that changes!)

Download Dialog

This dialog tells the user that software is available for download. The user can choose to begin the background update, postpone it for later, or never auto-download this particular package.

Netscape SmartUpdate

The new version of xyz is available.

Do you want to begin the update of this package by downloading it now?

Using continuous update, Navigator will download this package in the background while you aren't browsing the web or downloading something in the foreground.

Install Dialog

This dialog tells the user that software is available to install on the system. The user can choose to install the package now, wait until later, or never install it, causing it to be deleted from the system.

Preferences

The preferences for this feature are part of the Smart Update preferences panel.

Preference

Effect on Continuous Update

Enable continuous update

If continuous update is disabled, then users will still need to initiate software updates manually.

Default: On (?)

Require manual confirmation of each install

If this is enabled, then prompt the user once the software has been downloaded.

Default: On

Save directory

Downloads in progress will be downloaded to this directory.

Default: TBD

Frequency

How often to download the next chunk.

Default: TBD

Rate

How much data to get each time.

Default: TDB
