Summary

What is it?

It is a view side-by-side with the Navigator window that assists end users in finding 'the next bit of information' more quickly and easily. For Seamonkey the SideBar will have at most 2 content areas stacked on top of each other. The bottom area will deliver personalized news type content and may also be used as a special notification area, whereas the top area will deliver information/links related to the content the end users is currently browsing.

Goals

This is the first incarnation of a dynamic, flexible user interface that will be context sensitive, anticipate the users next steps and make exploring, browsing and searching for content on the network a less manual and more pleasant experience.

Target Users

The target users of Navigator 5 and specifically consumer types, that care less about underlying technology but more about getting results quickly and efficiently. This will also help beginning users get started and has the potential to be used in conjunction with accessibility designs later (since its requires less clicks and typing than conventional browsing methods)

Personalized Content/News/Notifcation Area

This is utilizing HTML as a technology to display web content that is broadcast from Netcenter. This content may be e.g. a subset of 'My/Our Netscape' content and will be personalized the same way, that is externally on Netcenter. 
 

SmartBrowsing Area

Will provide related links from Netcenter, but potentially also things like 'Related Books' from Amazon.com, related shopping etc. We may also include the three most recent search results in folders as well as site maps, rating, and security/privacy information about the site later. This list will be expandable after the product ships, since the data behind it is driven by Netcenter.

Issue:

When the user is offline we still may want to display meaningful data in the side bar, such as cached news/information. This may not work well for related links.

'My Netscape Live' Area

Has to be as simple to use as a TV:
  • Users will not be able to determine the ratio between HTML area and the SmartBrowsing area above (TBD: should we give them the ability to 'close' the HTML area and maximize SmartBrowsing?)
  • Even though the technology used is a poll technology initiated by the client, Users do not have to manipulate settings to get up-to-date and live information. Netcenter will set the site information and refresh interval remotely and store this information on the users machine.
Has to be as simple to configure as My Netscape:
  • We are expecting a web page on Netcenter to take car of customizing this area. It can be specified as a subset of My Netscape on the My Netscape page.

Opening and Closing the SideBar


By default the SideBar will come up opened. Closing can be performed by either clicking on the grippy area (works like 4.5 messenger), clicking on the toolbar icons that represents the SideBar or by (TBD) clicking on a 'close' command in the command area above the content. Re-opening can be performed by clicking on the closed grippy area (which will be only a few pixels wide visually, but has a wider mouse target area.) or by clicking the side bar icon on the toolbar again.

Issue:

Having the sidebar docked will probaly only work really well when the user has a 800*600 screen as a minimum (as of this writing almost 100% of PCs and Macs are sold with screens of 800*600 or 832*624 minimum respectively). What should we do about legacy and portable machines that only provide a VGA-sized screen (640*480)? There the Netcenter page would get clipped on the left side.

Accessing Links

Links are accessed using a single click. Despite the fact that we have sophisticated Aurora technology underneath a related links display, the list of related links will visually resemble 'real' links on a web page to afford user learned behavior when access related sites. The links should be underlined, and the text hilites when mousing over the link.
Clicking on a folder will toggle the open/close state of a folder. TBD: We should probably close up other folders not in the directly path of the currently opened folder in order to conserve vertical space. This would work like some of the Javascript folder users have encountered on web pages.
Clicking on a Link will always open it in the currently active content area. If the currently active content area contains frames, the link will replace the entire document. Some special links (such as help or about... links) may open into a new window.
 

Visual Design

Visually the smart browsing area should be part of the chrome, not the page. This is to avoid goofy looking situations where the page will provide it's own table of contents (such as news.com) that would then be wedged between a page like side bar and the real content page.

Issue:

The design of the page needs to be such that it does not distract from viewing the content, such that people would close the sidebar (thus defeating the revenue source that it can provide by tying end-users to Netcenter). We recommend low contrast colors when not being moused over. An idea being thought about is to move it to the right side of the screen to reduce the noise factor)
