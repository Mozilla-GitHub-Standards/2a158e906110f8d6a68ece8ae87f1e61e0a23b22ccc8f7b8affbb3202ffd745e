External Routes

Provide the traditional OS ways to access components like they are separate apps.

  • Windows Quicklaunch Bar- Offer Shortcuts to Nav, Mail, and IM
  • Desktop Shortcuts - Nav, Mail, IM
  • Windows Start Menu - during install, offer user to set Navigator/Mail as default Browser/Mail app - place shortcuts in Start Menu
    • Use *Installation Courtesy* - allow user to opt out of desktop shortcut placement, and OS default browser/mail setting.
    • Installer Specification TBA
  • Systray Icon - provide shortcut to Nav, Mail, and IM in a flyout menu, when quicklaunch is enabled
  • [Future] - Mac could use Application Bar for quicklaunch access and Component Shortcuts.






Internal Routes

  • Change Menu name and location - dependent on Menu Reorg work: from "tasks" to "windows" + "tools"); see Prelminary Menu Reorg Proposal "B"
  • Add icons to components in Tools menu (use current taskbar icons)
  • Create a new toolbar to the left of the Personal Toolbar: The component bar
  • Component Bar should contain (in order): Home, Mail, AIM (and/or ICQ), My Netscape, Search, Shop
  • Add new collapsing functionality to the component bar which allows user to grab and resize by draging a grippy. snapping would occur at 5 snapped intervals:
    1. fully open phase
    2. concatenation phase (to 3 letters, 4 letter words stay intact)
    3. no text phase (begin providing tooltips)
    4. one-by-one phase
    5. fully closed phase
        • concatenation would occur so that 4 letter words would remain intact. words greater than 4 letters would concatenate to 3. continued resizing would snap to "no text"phase.
        • from no text phase, continuing to resize would allow icons to fall off snapping to place once letting go, one by one, until the component bar is fully closed.
  • The former location for component icons becomes a "Tools" flyout menu, which contains component icons and menu options. This menu would persist in all component windows.
