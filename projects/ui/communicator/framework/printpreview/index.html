Design Objectives

  • feature used by all - beginner and expert
  • needs to be easy to exit
  • prevent loss of data or state
  • easy to implement in encroaching Mach V timeframe
  • 4.x concerns






Feature Objectives

  • WYSIWYG view - print based pre-visualization of screen based web content.
  • Final check to for user to ensure web content will conform to selected print media format
  • Page Breaking - ensure that number of pages for print job is desireable:
    • to avoid wasting paper
    • to ensure page breaking is appropriate to content
  • Visualization of page formatting - media proportions, margins, header and footer
    • shortcuts to format editing would be desireable secondary objectives






Essential Functions

  • Print...
  • [Page Setup] Shortcut
  • Cancel Print Preview
  • Doc Navigation
    • Jump to page [#]
    • Page Up/Down
    • Home/End
    • Pan - using space+click mousing
  • Print Scaling (manual scale to fit) - consider placing this feature in page setup
  • Orientation - consider placing this feature in page setup






Extras (if implementable)

  • +/- Zoom
  • %age Zoom box values
  • Print Scaling - (as a means to fit to page)
  • Background Color / Background Image Toggle on/off [x]
  • "On the fly" Printing Type size adjustment
  • Fit to Page
  • Fit to # of Pages
  • Orientation Shortcut - Landscape / Portrait
  • Header / Footer Editing Shortcut (takes you to Page Setup Dialog)






Visual Design

  • Print Preview window area should use a 55% black dithered gray background, achieved by using 70%/40%/40%/70% black checker pattern. Hex values: #999999, #4D4D4D

  • 1 pixel 100% black line page borders
  • 4 pixel offset 80% black (#333333) drop shadow for each page
  • Viewport margins: calculate a 130% page axial viewport margin for top and left






Description of Print Preview

Upon selection of the [Print Preview...] menu item, located under both the toolbar Printer combo button and in the File Menu directly between the [Page Setup...] command and the [Print...] command, the current browser window should transform into a chromeless view port containing the web content document in WYSIWYG print view, aided only by a print preview toolbar at the top. Using a chromeless window rather than a browser window based "print preview mode" (a' la Opera), was decided after considering the small amount of real space available to 800x600 users, and space to the print preview toolbar, should the sidebar be open.

The print preview toolbar should use the following layout:

Print... Page Setup... [ -- Viewport Navigation -- ] [ -- Zooming (n/a) --] [ Orientation ] [ Scaling to fit page ] [ future ] Cancel (right align)
  • File bugs as necessary for icons (using keyword "Icon")
  • there is a potential data loss with morphing the current window into print preview - if the user attempts to cancel print preview by closing the window, he/she also loses the current web page (this happened to Bill). We should throw a warning which alerts the user that closing the current window will lose the active web page, and to use the cancel button on the toolbar to quit print preview:
    Closing this window will abandon the current web page. If you do not wish to lose this page, next time use the Cancel button on the toolbar to exit Print Preview mode.

    Click Continue to exit Print Preview mode now, and return to the current web page
