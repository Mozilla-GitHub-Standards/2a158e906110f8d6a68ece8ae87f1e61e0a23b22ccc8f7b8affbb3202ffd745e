Goals

  • offer a design that is not overwhelming yet offers access to controlling the behavior
  • make it simple to get the best default
  • distinguish between the specification of how to handle the given instance of the content type they are currently downloading and the default handling of that file type
    • In this design the preference based "Edit File Type" is always being used to edit the default, whereas temprary handling of a file is handled within this first dialog.

When downloading a file

Case A: File type has been encountered before or has a know OS file type association

Once a filetype is has an entry in the helper apps preferences or it has an association that can be retrieved from the operating system and a file of the same type is downloaded, the dialog enables two main choices:

  • Use default action...: Using the default action that has been previously set up. This section will show the user in a text field what action has been set up as a default. The user can change that default setting again at any time a file is downloaded by using the "Change Default..." button or by going to the "Helper Applications" preferences to open the "Edit File Type" dialog. A checkbox can be unchecked to not show this dialog anymore when enocuntering this type of file and just appy the default action
    • Pressing OK in this situation will dismiss the dialog and apply the default action. This is the case even if Netscape does not have an entry in its database, since it falls back to letting the OS handle the file using the OS's default association.
  • Use a different action for this file: The user may choose a different action for the file that is currently being downloaded. Using option allows the user to treat a specific file differently for one time without permanently changing the defaults. For example a user chooses to open a specific .doc file using "Notepad.exe" because he or she knows its not really of type "application/ms-word". The sub-options are to either save the file or to open it with a specific application. The application name field in "Open with application" displays the Application's file name and should be made editable for Linux and Win32 to let users type in a path to the application.

Clicking on Set Default... will open a new dialog that lets the user set the defaults for that MIME type:

The top area of this dialog lets the user adjust the description, the extensions associated with that MIMR type (TBD: Mac users may want to set type and creator codes here). The checkbox "Always use this MIME type when sending Mail attachments" lets users map this MIME-types to files of this type that get attached to Mail messages.(Note: this prefs really belongs into a general section, not under Navigator)

The bottom area of this dialog lets the user set the default action. 4 possibilities are available:

  • "Handled by Netscape": Lets Netscape open this file as an internal document. This is only enabled for file types that Netscape is prepared to handle inline, such as JPG, GIF, PNG, HTML, TXT etc.
  • "Opened" (see screenshot below): lets the user set this so the downloaded document will be directly opened with an application
  • "Saved to disk": this would be default
  • "Used with a plugin": This option lets the user choose a plugin to open this file. This is enabled when there are plugins available on the user's Netscape installation that can handle this file type.

This screenshot below shows what this dialog looks like when the user chooses the option "Open this file". Two choices are displayed:

  • "Using default application": Lets the Operating System handle opening of the file. If the OS has this file type registered, then the application assigned to by the OS will be passed this file to open. If not the user see the usual OS based dialogs to pick an application
  • Using...: Lets the user pick a default application to open this type of file in when downloaded in Netscape. Note: this is a local setting that does not override the system settings. The application name field displays the Application's file name and should be made editable for Linux and Win32 to let users type in a path to the application.

Completing this setup will get the user back to the dialog from before except that that dialog has been changed to reflect the default settings that the user just edited. (see below in next section)

 

 

File type is encountered for the first time and has no default OS application associated with it

When a filetype is encountered for the first time and has no OS file type association, the user will see a dialog that lets him or he set up what to do this file in the future. That directs him towards the "Edit file type" window from the Helper Applications preferences.

OK button is disabled until either a default is set or "Use a different action for this file" is chosen.

Note: the dialog is similar in design to that design above being used when a file type either has an OS default association or the file type has been saved in Netscape's file database. so the user recognizes it and can leverage his or her knowledge on how the dialog works.

 

When creating a new file type in preferences

Goals

  • leverage same design layout/wording as above so users can see and understand the connection

Special cases:

In certain cases the vendor of a helper app may desire to not have the user go through having to see the first dialog at all. This is the case e.g with streaming video applications such as from RealVideo. In those cases the system setting depicted below is usually set to not "confirm open after download" in Win32. We should "listen to" this setting and bypass the first dialog altogether. does MacOS does have a similar system setting in its Internet control panel that we should also listen to ("Processing with Application").

 

Notes

Possible assumptions that can be made to simplify choices:

- Never write-modify an OS setting, setting is only used in Netscape
- The Netscape file types database does not replicate the OS file associations, it only records when files are set up differently for use with Netscape.
- if this implimented in stages, the dialog could be initially released without plugin integration
