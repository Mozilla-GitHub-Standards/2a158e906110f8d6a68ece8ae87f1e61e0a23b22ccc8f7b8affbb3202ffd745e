Helper Apps User Interface | Proposal

German W. Bauer, Client UE     February 16, 2001

 

Goals

  • offer a design that is not overwhelming yet offers access to controlling the behavior
  • make it simple to get the best default
  • let the user distinguish between the specification of how to handle the given instance of the content type they are currently downloading and the handling of similar content type in the future

When downloading a file

Design A:

Presenting all choices at once

+ all choices immediately visible

- dialog is a bit overwhelming

Design B:

Selection based dialog using menu list

+ uncluttered design
+ easily extensible to other types of handling

- other choices may be difficult to discover

Design C:

Selection based dialog using radio buttons

+ uncluttered design

 

Possible assumptions that can be made to simplify choices:

- Saving a file is never remembered
- Never write-modify an OS setting, setting is only used in Navigator
- if this implimented in stages, the dialog could be initially release without plugin integration
- display the executable name instead of a description, easier to understand how that relates to choose...

When editing the file type in preferences

Goals

  • leverage same design layout/wording as above so users can see and understand the connection

Note:

On MacOS this dialog could be extended to give access to the OS's "File Exchange" control panel which allows users to map downloaded file to Mac files using proper creator type and document type.
