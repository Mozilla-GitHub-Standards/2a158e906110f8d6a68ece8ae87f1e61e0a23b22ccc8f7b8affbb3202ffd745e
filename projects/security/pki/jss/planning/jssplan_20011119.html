JSS Draft Plan for Winter-Spring 2001-2002

Jamie Nicolson
April 15, 2002
Primary Newsgroup: mozilla.dev.tech.crypto
Alternate Newsgroup: mozilla.dev.tech.crypto

The next versions of JSS will focus on compliance with Java standards. The actual version numbers will depend on whether backward compatibility is maintained.

This plan is a draft. Many aspects will be reviewed during implementation.

    JDK version

      Some customers are stuck on JDK 1.2, but several new features have been introduced in JDK 1.4 that we want to take advantage of. We will release two versions of the JSS jar file. One version ( Gravisaurus) will contain all the latest features, but will require JDK 1.4. The other version (Antiquusaurus) will work with JDK 1.2, but will not contain all the new features (JSSE and the cert path API).

Backward Compatibility

    As new interfaces are added, I will try to present all existing functionality through the new interfaces. Where this is possible, I will deprecate the old API. I don't see any need to actually remove the old methods/classes.

Schedule

The JSS features will be split between two feature releases.
