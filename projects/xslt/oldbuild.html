How to build Mozilla with XSLT?
To integrate XSLT in your build, you need to build mozilla/extensions/transformiix.

Unix
Add 
ac_add_options --enable-xsl
ac_add_options --with-extensions=default,transformiix
to your .mozconfig, and use client.mk to build as usual.

Macintosh
Set the TransforMiiX option flag in mozilla/build/mac/BuildNGLayout(Debug).pl to 1 (from "$options{transformiix} = 0;" to "$options{transformiix} = 1;"). Build and you're done.

Windows
Set the DOS environment variable MOZ_XSL You currently need to patch the mozilla/extensions/makefile.win to include the transformiix directory.

How to build the TransforMiiX standalone processor?
To build the standalone processor, you have to checkout the needed parts of the source and build.

Unix
On Unix the TransforMiix executable can be built without Mozilla.

To do so, add 

mk_add_options BUILD_MODULES=transformiix
ac_add_options --enable-modules=transformiix
to your .mozconfig, and use client.mk as usual. Do NOT add --enable-xsl. This will only check out and build those sources needed for transformiix. The generated object files for the Mozilla component and the standalone executable are different, so you must use a different build directory for the standalone, if you are using transformiix for Mozilla as well.

Windows
To build TransforMiix on Windows, see the Windows build page and pull 
mozilla/config
mozilla/expat
mozilla/extensions/transformiix
from CVS and 
set MOZ_BITS=32
cd mozilla\extensions\tranformiix
nmake /f makefile.win standalone
The binary will be in mozilla\extensions\transformiix\transformiix.exe.
