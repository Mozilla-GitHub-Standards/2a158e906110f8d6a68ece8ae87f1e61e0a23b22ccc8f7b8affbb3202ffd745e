TransforMiiX error codes

Axel Hecht

This page lists the error types occuring in the TransforMiiX xpath/xslt engine. Error codes with grey background are currently existing, white background indicates that error conditions like this exist. As of now, not a single of these is frozen.

Component Code Description
General
ERROR_INTERNAL Internal error
ERROR_MEMORY Out of memory
Lexer
ERROR_UNRESOLVED_VAR_REFERENCE A $ is not followed by a QName
ERROR_OP_EXPECTED Violation of bullet 1 of 3.7 Lexical Structure
ERROR_UNCLOSED_LITERAL 'foo or "bar
ERROR_COLON Colon must only be used in QNames, not foo:bar:baz
ERROR_BANG ! is only permitted as !=, negation is boolean function not()
ERROR_UNKNOWN_CHAR fallback error, neither a NAME character nor a recognized special character
Parser
ERROR_MISSING_RIGHT_BRACE Ending '}' missing in AVT
ERROR_EXPECTING_RIGHT_PARENTHESIS Unexpected token, right parenthesis expected
ERROR_UNRECOGNIZED_AXIS Unrecognized axis (really an internal error)
ERROR_UNEXPECTED_TOKEN Unexpected token
ERROR_EXPECTING_LEFT_PARENTHESIS Unexpected token, left parenthesis expected (we could get rid of this error)
ERROR_EXPECTING_LEFT_PARENTHESIS_OR_LITERAL Unexpected token, right parenthesis expected (or literal for pi)
ERROR_EXPECTING_RIGHT_BRACKET Unexpected token, right bracket expected
ERROR_EXPECTING_LEFT_PARENTHESIS_OR_COMMA Unexpected token, right parenthesis or comma expected
Expressions
ERROR_WRONG_NUMBER_OF_ARGUMENTS Wrong number of arguments to function
ERROR_NON_NODESET_ARGUMENT Expected nodeset as argument
ERROR_EXPRESSION_SHOULD_PRODUCE_NODESET All expression parts of a PathExpr should produce a nodeset
