webclient implementation guide

object hierarchies | sequence description | classes & interfaces


introduction

This document an engineering overview of the Mozilla wrapping implementation of Webclient that complies with version 1.7 of the requirements specification. It supplements comments in the source code.

The phrase custom app as used in this document refers to an application that (1) uses Webclient to add browsing capability to its feature set and (2) is written by a third party. At this time, all classes and interfaces exist in the current implementation, but many of the methods have not been implemented. Enough are implemented, however, to provide Webclient M2 functionality, as well as viewing of bookmarks and DocumentLoadListener and MouseListener events.

source directories and package hierarchies

This section explains the directory structure for the source code and what is located in each directory. All paths are relative to mozilla/java/webclient. Directories are examined depth-first.
 
classes
This is the root of the Java code for the pre-spec-compliant implementation. The code in classes doesn't concern this document. 
classes_spec
This is the root of the Java code for the spec-compliant implementation. The subdirectory structure of classes_spec mirrors Java packages, so code in classes_spec/org/mozilla/webclient is in Java package org.mozilla.webclient. 
classes_spec/org/mozilla/webclient
This directory contains the Webclient public interfaces, BrowserControlFactory, which is the entry point for users of the code, and a few classes used in sub-packages. 
classes_spec/org/mozilla/webclient/test
This directory contains the EmbeddedMozilla test browser. 
classes_spec/org/mozilla/webclient/test/swing
This directory contains the SwingEmbeddedMozilla test browser. This code hasn't been rev'd to use the spec-compliant implementation. 
classes_spec/org/mozilla/webclient/wrapper_native
This directory contains the Java code for wrapping a native browser. There is no browser-specific Java code in org.mozilla.webclient. All browser-specific code for wrapping native browsers is contained in the JNI implementation code. This directory contains Java classes that define native methods which must be implemented by the browser provider. For example, the code in classes_spec/org/mozilla/webclient/src_moz provides an implementation for these native methods for the Mozilla browser. 
classes_spec/org/mozilla/webclient/wrapper_native/motif 
This directory contains Java code for wrapping a native browser on X. 
classes_spec/org/mozilla/webclient/wrapper_native/win32
This directory contains Java code for wrapping a native browser on win32.
classes_spec/org/mozilla/webclient/wrapper_nonnative
This directory will contain code for wrapping a Java-based web browser. 
src
This is the root of the native code for the pre-spec-compliant implementation. The code in src doesn't concern this document. 
src_ie
This directory will contain code for the IE implementation of the JNI interfaces in classes_spec/org/mozilla/webclient/wrapper_native .
src_moz
This directory contains code for the Mozilla implementation of the JNI interfaces in classes_spec/org/mozilla/webclient/wrapper_native. 
src_moz/motif 
This directory contains the Unix-specific code for the Mozilla implementation of the JNI interfaces.
