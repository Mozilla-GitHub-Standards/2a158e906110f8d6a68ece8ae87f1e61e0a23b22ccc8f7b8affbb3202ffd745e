Localization Engineering Check List

by Tao Cheng




In addition to the dtd/properties files under chrome/[package]/locale/[locale-name], there are certain files need to be localized to represent a Browser client's localization characteristics.

This table lists all the items and the appropriate values for a Japanese builds. Please refer to the Communicator 4.x for values suitable to other languages.
 
File Name Value for Japanese Build Description check
chrome/[package]/locale/*.{dtd, properties} (UI strings) Strings appear in UI  
All chrome/[package]/locale/chrome.rdf The <name> attribute ja-JP indicate which locale directories to load UI strings from  
Preferences in defaults/pref/all.js general.useragent.locale ja-JP the locale info in the user-agent string  
intl.accept_languages ja the "Accept-Language" field of the HTTP header  
intl.charset.default Shift_JIS the Browser's default charset  
intl.charset_menu.static iso-2022-jp, shift_jis,euc-jp,iso-8859-1  
intl.charset.detector japsm auto charset detection of incoming document  
Preferences in defaults/pref/mailnews.js mailnews.send_default_charset iso-2022-jp  
mailnews.view_default_charset iso-2022-jp
