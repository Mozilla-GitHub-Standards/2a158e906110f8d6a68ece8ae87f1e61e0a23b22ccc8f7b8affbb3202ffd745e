Customization Freeze


Background

Client Customization Kit (CCK) relies on knowing the exact location of the resources to customize the browser client. Should the location of such resources moved within the files or across files, CCK might not work. Therefore, when we are close to a planned release date, we'd like to restrict changes to those resources and we can control the quality of CCK and ship it on schedule.

What files are subject to this restriction?

Here is a list of files CCK modifies:
  1. Basically, all localizable resources are customizable. This includes files to be packaged into deflenus.xpi, regus.xpi and langenus.xpi.
  2. All preference values ([install-dir]\defaults\pref\*.js) 
  3. In browser.xpi,
    • bin/chrome/toolkit.jar
      • global/content/helpMenu.xul 
      • bin/chrome/classic.jar
        • skin/classic/communicator/brand/throbber-anim.gif   (locking file path only)
        • skin/classic/communicator/brand/throbber-single.gif (locking file path only)
    • bin/chrome/modern.jar
      • skin/modern/communicator/brand/throbber-anim.gif  (locking file path only)
      • skin/modern/communicator/brand/throbber-single.gif  (locking file path only)
    • bin/chrome/comm.jar
      • content/communicator/communicatorOverlay.xul

What to do if you'd like to modify these files

Please get approval from Customization Engineering Team before checking in the patches.
