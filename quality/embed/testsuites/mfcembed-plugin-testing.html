Adobe Acrobat Reader
  1. Download latest Adobe Acrobat Reader
    1. Launch Installer and proceed to install.
    2. Test Acrobat Reader Functionality by going to http://www.irs.gov/forms_pubs/forms.html and try to view .pdf files
  2. If installer fails to detect mfcembed embed\plugins\ directory:
    1. Make sure Netscape 6.1 or latest Mozilla is installed
    2. Launch the Acrobat Reader installer and proceed to install
    3. Locate nppdf32.dll
      1. In Netscape 6.1, go to the Plugins\ directory and copy nppdf32.dll to the embed\plugins\ directory, exit/close Netscape and restart mfcembed to load the plugin
      2. In Mozilla, go to the Plugins\ directory and copy nppdf32.dll to the embed\plugins\ directory, exit/close Mozilla and restart mfcembed to load the plugin
      3. Delete or rename nppdf32.dll in Netscape or Mozilla plugin location
      4. Test Acrobat Reader Functionality by going to http://www.irs.gov/forms_pubs/forms.html and try to view .pdf files using mfcembed
  1. ar500enu.exe downloaded.  Adobe Acrobat Installer should detect mfcembed build and plugin directory via win32 registry key entry.  Should be able to view .pdf files within mfcembed.
  2. Need to manually place .dll(s) in mfcembed embed\plugins\ directory
 1. The Acrobat Reader installer doesn't detect the mfcembed build.  Unable to open .pdf files within mfcembed. 

***Contacts at Adobe need to be made aware of new Win reg key(s) for all Gecko based browsers. Bugzilla bug 77244***

2. Manually placing the latest nppdf32.dll doesn't work if acrobat reader application isn't already installed. 
If acrobat is installed and .dll is moved to mfcembed embed\plugins\ user can load .pdf files inline.
