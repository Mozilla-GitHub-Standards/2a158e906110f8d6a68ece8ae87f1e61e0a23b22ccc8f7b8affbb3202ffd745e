Test Plan for nsIFontList interface

Introduction:
The nsIFontList interface provides an application the necessary information so that the user can select the font to use as the default style sheet. This is used if the style sheet is missing or does not specify a font.
Font lists are specified per language group.
The Test case also provides the browser information such as the application name, user agent, code name, app version, language, platform.

Approach:
These test cases are written in JavaScript embedded in HTML. They can be run directly or from ngdriver(ngdriver is server that host the testcases).
If you run the test cases directly the results are reported on the same html page. If you run the test cases from ngdriver the results are saved into a file so that you can refer to it later.

You can find the source code of test cases at
http://lxr.mozilla.org/seamonkey/source/embedding/qa/jstests/nsIFontListTest.txt
When you download the test case rename the text file to .html file before using it.

To run the test cases directly open nsIFontListTest in Embedding test application or click on the link below.
http://mozilla.org/quality/embed/jstests/nsIFontListTest.html
Note: Before executing the test case be sure to add the following line to your prefs.js file.
user_pref("signed.applets.codebase_principal_support", true);

To run test cases from ngdriver follow the steps below:
[Note: Right now you can run the tests from ngdriver only if you have access to bubblegum server].
1. Type the url http://bubblegum/ngdriver/tester.html in Embedding test application url location.
2. Replace suites/suite1 with suites/embedding and click on "Find Testcases in:". You will see the refreshed window with test cases listed in the embedding suite.
3. Give a name for the test in the text box "Specify a NAME for this test run:". Then select nsIFontListTest.html
    and click on "Run Testcase". This will run the test case and it will provide a link to the results file.
4. Click on resultsfile.html to view the results.

Features Tested:
Method: availabeFonts

Introduction:
Gets the list of available fonts for a language group and for use as the given CSS generic font.

Syntax:
nsISimpleEnumerator availableFonts (in wstring aLangGroup, in wstring aFontType);

aLanguageGroup limits the fonts to fonts in a language group;
e.g.: x-western (American/Western European),   ar (Arabic),  el (Greek),  he (Hebrew),  ja (Japanese),
       ko (Korean), th (Thai), tr (Turkish), x-baltic (Baltic), x-central-euro (Eastern European),
       x-cyrillic (Russian), zh-CN (China), zh-TW (Taiwan).

aFontType limits the fonts to the fonts with this CSS generic font type;
e.g.: serif, san-serif, cursive, fantasy, monospace.

availableFont returns a simple enumeration of the available fonts for a language group/ generic type. These strings are the text supplied by the operating system's font system.
