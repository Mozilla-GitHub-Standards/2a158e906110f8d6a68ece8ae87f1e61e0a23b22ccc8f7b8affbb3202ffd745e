EmbedSmokeTest Test Plan

Introduction:
EmbedSmokeTest is an automated test case that can be used to perform smoke test of Embedding test applications like mfcEmbed.

Approach:
These test cases are written in JavaScript embedded in HTML. EmbedSmokeTest can be run directly or from ngdriver (ngdriver is server that hosts the testcases).
If you run the test cases directly the results are reported on the same html page. If you run the test cases from ngdriver the results are saved into a file so that you can refer to it later.

You can find the source code of test cases at
http://lxr.mozilla.org/seamonkey/source/embedding/qa/jstests/EmbedSmokeTest.txt
When you download the test case rename the text file to .html file before using it.

To run the test cases directly open EmbedSmokeTest in Embedding test application or click on the link below.
http://mozilla.org/quality/embed/jstests/EmbedSmokeTest.html

To run test cases from ngdriver follow the steps below:
[Note: Right now you can run the tests on ngdriver only if you have access to bubblegum server].
1. Type the url http://bubblegum/ngdriver/tester.html in Embedding test application url location.
2. Replace suites/suite1 with suites/embedding and click on "Find Testcases in:". You will see the refreshed window with test cases listed in the embedding suite.
3. Give a name for the test in the text box "Specify a NAME for this test run:". Then select embedSmokeTest.html and click on "Run Testcase". This will run the test case and it will provide a link to the results file.
4. Click on resultsfile.html to view the results.

Features Tested:
Following table shows detailed description and Pass/Fail criteria of each test case. More test cases will be added to EmbedSmokeTest and the table below will be updated accordingly.
 
 

No: Test Description  Pass Criteria Fail Criteria
1 Link Clicking Link is automatically clicked on a page. A new window is opened with the specified href. Once the new window is loaded completely pass/fail criteria is set and the window is closed. If the clicking of the link was successful and if the new window is opened with the href successfully the result table will show "Pass". If the clicking of the link was unsuccessful and if the new window does not open successfully the result table will show "Fail".
2 URL Loading Opens a new window and set the url of the window. (say to http://www.mozilla.org"). Checks whether the url is loaded sucessfully and checks whether the scrollbars appear properly. If the new window with url is loaded properly and if the scrollbars appears properly the result table will show "Pass". If the new window with url is not loaded properly or if the scrollbars doesnot appear properly the result table will show "Fail".
3 File URL Loading Opens a new window and set the location bar to load a html file. Checks whether the file is loaded properly and contains content. If the new window with the specified file opens properly and if it contains the content the result table will show "Pass". If the new window with the specified file doesnot open properly or if it doesnot render the  content the result table will show "Fail".
4 Loading of GIF Image. Ensures that the GIF image is loaded. If the GIF image is loaded properly, the results table will show "Pass". If the GIF image is not loaded the results table will show "Fail" with red background.
5 Loading of JPEG Image. Ensures that the JPEG image is loaded. If the JPEG image is loaded properly, the results table will show "Pass". If the JPEG image is not loaded properly, the results table will show "Fail" with red background.
6 Directory URL Loading [Not complete yet] Opens a new window with the directory listing.Checks whether the directory is loaded properly and contains content. If the new window with the specified directory opens properly and if it contains the content the result table will show "Pass". If the new window with the specified directory doesnot open properly and if it doesnot show all the files in that directory the result table will show "Fail".

Do you want to add test cases to EmbedSmokeTest ?
Follow the steps below if you want to add your test cases to EmbedSmokeTest.
1. First add the checkbox.
    Within the body there is script [if(readCookie("embedsmoketest") == null),], add your checkbox in this script like
    document.write("<input type='checkbox' name='checkboxname'><b> Your text </b></input><br>");
2. Put your logic in the function onSubmit().
    eg:  if( (document.form1.checkboxname.checked==true) || (executeAllTestCases=="true") )
          {  your code goes here.    }
3. Append your results at the end of the function constructResults().
