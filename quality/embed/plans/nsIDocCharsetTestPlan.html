Test Plan for nsIDocCharset interface

Introduction:
The nsIDocCharset interface allows the user to view, hint, and set which converter is used to read the document's data.
Inside Mozilla all text is encoded as Unicode. When the document is read in, a converter is used to convert the document's data to Unicode.
nsIDocCharset provides the ability to:
See what converter was used when inputting the documents data.
Override the character set for documents where the specified fallback, or auto-detected character set is incorrect.

Approach:
These test cases are written in JavaScript embedded in HTML. nsIDocCharsetTest can be run directly or from ngdriver (ngdriver is server that hosts the testcases).
If you run the test cases directly the results are reported on the same html page. If you run the test cases from ngdriver the results are saved into a file so that you can refer to it later.

You can find the source code of test cases at
http://lxr.mozilla.org/seamonkey/source/embedding/qa/jstests/nsIDocCharsetTest.txt
When you download the test case rename the text file to .html file before using it.

To run test cases directly open nsIDocCharsetTest in Embedding test application or click on the link below.
http://mozilla.org/quality/embed/jstests/nsIDocCharsetTest.html
Note: Before executing the test case be sure to add the following line to your prefs.js file.
user_pref("signed.applets.codebase_principal_support", true);

To run testcases from ngdriver follow the steps below:
[Note: Right now you can run the tests from ngdriver only if you have access to bubblegum server].
1. Type the url http://bubblegum/ngdriver/tester.html in Embedding test application url location.
2. Replace suites/suite1 with suites/embedding and click on "Find Testcases in:". You will see the refreshed window
with test cases listed in the embedding suite.
3. Give a name for the test in the text box "Specify a NAME for this test run:". Then select nsIDocCharsetTest.html
and click on "Run Testcase". This will run the test case and it will provide a link to the results file.
4. Click on resultsfile.html to view the results.

Attributes Tested:
charset
 
No: Test Description Pass Criteria Fail Criteria
1 Initial charset value This will get the current encoding used to read the document. You can find this by selecting view->Character coding.The dot will tell you what character coding is currently used. If it is able to list the current encoding used then the result table will show "Pass". If it is not able to list the current encoding used then the result table will show "Fail".
2 Charset value after setting.
Note:Whatever charset you want to set, you can do it by changing setCharsetValueTo variable in the test case.

var setCharsetValueTo = "ISO-8859-1";

 This test case will set the charset to whatever you specified in setCharsetValueTo variable and reloads the document to acctually have its effect. You can check this after running the test case by selecting view->Character coding.you can notice that it is changed to whatever you specified. If it is able to set the charset to whatever you specified then the result table will show "Pass". If it is not able to set the charset to whatever you specified then the result table will show "Fail".
