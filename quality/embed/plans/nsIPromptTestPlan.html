Test plan for nsIPrompt interface

    nsIPrompt interface provides methods to allow the client to implement its own version of the dialogs that
    Gecko may need to display such as when a user goes from a secure to a non-secure page, or when page
    loading times out.

    In order to get nsIPrompt object, you need to first get a window watcher object.

    var ww = Components.classes["@mozilla.org/embedcomp/window-watcher;1"].
                                       getService(Components.interfaces.nsIWindowWatcher);

    Once you get window watcher object you can get nsIPrompt object by calling getNewPrompter method
    on window watcher.

    // window is javascript window which is always available.
    var prompter = ww.getNewPrompter(window);  // nsIPrompt

    Once you get prompt object you can call the methods of nsIPrompt interface.

    To see the test cases click on the desired methods below.

    Click here to see the source code for the test case. If you download the testcase rename it to .html before opening
    it in an embedding application.

    alert()
    alertCheck()
    confirm()
    confirmCheck()
    confirmEx()
    prompt()
    promptPassword()
    promptUsernameAndPassword()
    select()

confirmEx()

    Introduction:
    Puts up a dialog with upto 3 buttons and an optional checkbox.

    How to use:
    Buttons are numbered 0 to 2. The sequence whether it goes from left to right or right to left depends on the environment.

    First Variation:
    var checkbox = {value:0};
    var buttonPressed = {value:0};
    var std_buttons = (promptObj.BUTTON_TITLE_OK * promptObj.BUTTON_POS_0) +
                               (promptObj.BUTTON_TITLE_CANCEL * promptObj.BUTTON_POS_1) +
                               (promptObj.BUTTON_TITLE_SAVE * promptObj.BUTTON_POS_2);
    promptObj.confirmEx("confirmEx dialog", "text message", std_buttons,
                                      "Button0", "Button1", "Button2",
                                      "check or uncheck", checkbox, buttonPressed);
    alert("Checkbox checked: " + checkbox.value);
    alert("Button pressed: " + buttonPressed.value);

     Second Variation:
     Just replace the std_buttons assignment with this.
     var std_buttons = (promptObj.BUTTON_TITLE_IS_STRING * promptObj.BUTTON_POS_0) +
                                (promptObj.BUTTON_TITLE_IS_STRING * promptObj.BUTTON_POS_1) +
                                (promptObj.BUTTON_TITLE_IS_STRING * promptObj.BUTTON_POS_2);
     If you use the above statement you should notice that predefined title is no longer used. All the three buttons come up with
     titles  "Button0", "Button1", "Button2".

     Below test cases are w.r.t First Variation.
 

Case Steps Expected Result Pass/Fail Comments
1 Use the above code. Dialog should come up with 3 buttons.OK, Save, Cancel. pass The buttons use the predefined titles specified by the constants. It ingores the names you give.If you want your titles to come up use promptObj.BUTTON_
TITLE_IS_STRING.
2 Click the OK Button. Button Pressed should return 0. pass 0 is the index of the first button.
3 Click the Save Button. Button Pressed should return 2. pass
4 Click the Cancel Button Button Pressed should return 1. pass
5 Try with Other predefined titles. Should return indexes depending on the position of the buttons and button titles should come up correspondingly. pass
6 Check Checkbox.
Click OK.		 Checkbox checked should return true.
Button pressed should return 0.		 pass
7 Check Checkbox.
Click Cancel.		 Checkbox checked should return true.
Button pressed should return 1.		 pass
8 Check Checkbox.
Click Save.		 Checkbox checked should return true.
Button pressed should return 2.		 pass
9 Make the checkbox value 1 instead of 0. The dialog should come with the checkbox checked. pass Since its an inout parameter.

select()

    Introduction:
    Puts up a dialog which has a list box of strings.

    How to use:
    selectList = new Array("first", "second", "third", "fourth");
    var outSelection = {value:0};
    var boo = promptObj.select("select dialog", "Select one:",
                                                 4, selectList, outSelection);
    alert("outSelection: " + outSelection.value);
    alert("confirmation accepted: " + boo);
 

Case Steps Expected Results Pass/Fail Comments
1 Select an item in the list. Outselection should contain the index of the item when you click OK.If you click Cancel Outselection will contain the index 0(default). pass
