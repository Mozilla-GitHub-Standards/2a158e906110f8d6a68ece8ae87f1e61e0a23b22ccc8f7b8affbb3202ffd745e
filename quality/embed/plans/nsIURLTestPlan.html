Test Plan for nsIURL interface

   The nsIURL interface provides convenience methods that further break down the path portion of nsIURI.
    In order to test the functions of  nsIURL you need to first get an instance nsIURL.

    var nsUrlObj = Components.classes["@mozilla.org/network/standard-url;1"].
                                                createInstance(Components.interfaces.nsIURL);

    Now you can call the methods using the uriObj object.

    To see the test cases click on the desired method or attributes below.

    Click here to see the source code for the test case. If you download the testcase rename it to .html before opening
    it in an embedding application.

   Attributes:
    filePath
    param
    query
    ref
    directory
    fileName
    fileBaseName
    fileExtension
    escapedQuery

filePath

    Introduction:
    Returns the path including the directory and file portions of a URL.

    How to use:
    urlObj.spec = "http://directory/newdir/fileBaseName.fileExtension;param";
    alert(urlObj.filePath);
    urlObj.filePath = "/modifieddir/modifiedfile.ext";
    alert(urlObj.spec);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. If spec is
"http://directory/newdir/
fileBaseName.fileExtension;param"
then the alert for filePath should return
"newdir/fileBaseName.fileExtension".		 pass
2 Set the filepath to
"/modifieddir/modifiedfile.ext".		 The alert for spec should return 
"http://directory/modifieddir/
modifiedfile.ext".		 pass

param

    Introduction:
    Returns the parameters specified after the ; in the URL.

    How to use:
    urlObj.spec = "http://directory/fileBaseName.fileExtension;param";
    alert(urlObj.param);
    urlObj.param = "newparam";
    alert(urlObj.spec);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. If spec is 
"http://directory/fileBaseName.
fileExtension;param" then the alert
for param should return "param".		 pass alert for param return whatever is there after ";".
2 Change the urlObj.spec to "http://directory/fileBaseName.
fileExtension?query".		 Should return null since there is no ";" in the path. pass
3 Set the param to "newparam". The alert for spec should return 
"http://directory/fileBaseName.
fileExtension;newparam".		 pass
