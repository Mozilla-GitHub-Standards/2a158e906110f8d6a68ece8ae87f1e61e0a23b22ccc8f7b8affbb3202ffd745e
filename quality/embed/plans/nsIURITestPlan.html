Test Plan for nsIURI interface

    URIs are essentially structured names for things. This interface provides accessors to destructure those names.
    In order to test the functions of nsIURI you need to first get an instance nsIURI.

    var uriObj = Components.classes["@mozilla.org/network/standard-url;1"].
                                                createInstance(Components.interfaces.nsIURI);

    Now you can call the methods using the uriObj object.

    To see the test cases click on the desired method or attributes below.

    Click here to see the source code for the test case. If you download the testcase rename it to .html before opening
    it in an embedding application.

    Attributes:
    spec
    prePath
    scheme
    preHost
    username
    password
    host
    port
    path

    Methods:
    equals()
    schemeIs()
    clone()
    resolve()

spec

    Introduction:
    Returns a string representation of the URI. Setting the spec causes the new spec to be parsed, initializing the URI.
    Settting the spec causes also the currently open streams on the URI's channel to be closed.

    How to use:
    // Before you alert the attribute spec you somehow have to set it. In current case I set it by assigning a string.
    uriObj.spec = "http://username:password@www.mozilla.org:9090/quality/embed/index.html";
    alert(uriObj.spec);
 

Case Steps  Expected Results Pass/Fail Comments
1 Use the above code. It should alert the spec. pass

port

    Introduction:
    Returns the port of the URI. A return value of -1 indicates that no port value is set and the implementor of the specific
    scheme will use its default port. Similarly setting a value of -1 indicates that the default if to be used. For HTTP, Port
    80 is same as a return value of -1.

    How to use:
    alert(uriObj.port);
    uriObj.port = 9595;
    alert(uriObj.port);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. If spec is "http://username:password@www.
mozilla.org:9090/quality/embed/index.html" then the alert for port should return 9090.		 pass
2 Remove the port from the spec. If spec is ""http://username:password@www.
mozilla.org/quality/embed/index.html"
then the alert for port should return -1.		 pass
3 Set port to 9595. The alert for port should return 9595. pass
4 Set port to -1. The alert for port should return -1. pass Setting port to -1 indicates that default is to be used.

equals()

    Introduction:
    Compares two URIs. Note that this comparision is only on char* level. For example in HTTP:
    http://foo.com:80 == http://foo.com. but this function through nsIURI alone will not return equality for this case.

    How to use:
    First Variation:
    uriObj.spec = "http://username:password@www.mozilla.org:9090/quality/embed/index.html";
    newuriObj.spec = "http://username:password@www.mozilla.org:9090/quality/embed/index.html";
    var boo = uriObj.equals(newuriObj);
    alert(boo);

    Second variation:
    uriObj.spec = "http://username:password@www.mozilla.org:80/quality/embed/index.html";
    newuriObj.spec = "http://username:password@www.mozilla.org/quality/embed/index.html";
    var boo = uriObj.equals(newuriObj);
    alert(boo);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. (First variation). The alert should return true since both the URIs are same. pass
2 Remove the path of newuriObj. (First variation). The alert should return false since the URIs are not equal. pass
3 Use the above code. (Second variation). The alert should return false since both the URIs are not equal. pass Note that this comparision is only on char* level. Though in HTTP these two are equals this function alone will not return equality for this case.

schemeIs()

    Introduction:
    An optimization to do the scheme checks without requiring the users of nsIURI to GetScheme thereby saving extra
    allocating and freeing. Returns true if the schemes match (case ignored). Note that for unknown cases this will always
    return false.

    How to use:
    var boo = uriObj.schemeIs("http");
    alert(boo);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. If spec is 
"http://username:password@www.
mozilla.org:9090/quality/embed/index.html"
then the alert for schemeIs should return true.		 pass Since both the schemes are same. "http".
2 Change the input string to "ftp".(ie. var boo = 
uriObj.schemeIs("ftp").		 If spec is 
"http://username:password@www.
mozilla.org:9090/quality/embed/index.html"
then the alert for schemeIs should return false.		 pass Since the schemes are not same.

resolve()

    Introduction:
    This method resolves a relative string into an absolute URI string, using the URI as the base.

    How to use:
    var absuri = uriObj.resolve("/projects/ui/accessibility/");
    alert(absuri);
 

Case Steps Expected Results Pass/Fail Comments
1 Use the above code. If spec is 
"http://username:password@www.
mozilla.org:9090/quality/embed/index.html"

mozilla.org:9090/projects/ui/accessibility/.		 pass
2 Remove the starting "/" in the relative path. The path should be added at the end.
If spec is 
"http://username:password@www.
mozilla.org:9090/quality/embed/index.html"
then the absuri should return "http://username:password@www.
mozilla.org:9090/quality/embed/projects/
ui/accessibility/.		 pass
