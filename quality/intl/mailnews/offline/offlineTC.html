Test Cases


Test Case # Objective Steps To Perform Expected Results Comments
Offline01 Non-ascii mail and news folders in Items for Offline Use dialog 1. Open Mail.
2. Go to File > Offline > Download/Sync Now, click Select button, Items for Offline Use dialog appears.		 The non-ascii folder and sub folder names display properly.  
Offline02 Downloading non-ascii mail folders 1. Open Mail.
2. Go to File > Offline > Download/Sync Now, click Select button.
3. Click checkbox next to non-ascii mail folders.
4. Click OK (then wait for synchronization to complete).
5. Go to File > Work Offline.
6. Try to access the messages in a folder you selected for offline usage.		 1. Non-ascii folders are download during sync.
2. The contents of non-ascii folders are accessible when offline.
3. Non-ascii subjects are displayed properly.
4. Non-ascii characters in body of message are displayed properly.
5. Non-ascii sender is displayed properly. 
6. Non-ascii attached file name is displayed properly.
7. Non-ascii inserted link, table, anchor name, HTML, chacters & marks are displayed properly.		  
Offline03 Character coding selection after synchronization 1. Go to File >Offline> Work Offline.
2. Create a new message.
3. Go to View > Character Coding, select a character coding other than the default setting in Preferences.
4. Save the message as a draft.
5. Go back online, then go to File > Work Offline > Download/Sync Now.
6. Open the message created.		 The character coding setting is consistent after synchronizing.  
Offline04 Send Later from icon 1. Open Mail
2. Go to File > Offline > Work Offline 
3. Compose a Non-ascii message with subject,
attached file name, attached file,  inserted
link name, table contents, anchor name, HTML, characters&marks.
4. Select View > Character Coding  appropriate     character coding from menu. 
5. Click Send Later icon *		 The composed message shouldn't be sent and it can be saved into Unsent Message folder properly.

* Non-ascii subjects are display properly.
* Non-ascii characters in body of message are display properly.
* Non-ascii attached file name is displayed properly.
* Non-ascii inserted link, table, anchor name, HTML, chacters & marks are displayed properly.
Offline05 Send Later from menu Excute Offline04
Do the following step instead of step 4 above
5. Select File >  Send Later 		 Same as Offline04
Offline06 Send Later Messages 1. Open Mail
2. Go to File > Offline > Work Offline 
3. Compose a non-ascii message with non-ascii subject, non-ascii Sender, non-ascii recipiant
4. Click Send Later icon or menu
5. Click OK button *
6. Repeat step 1. and 2.
7. Click Cancel button then back to Compose window.**
8. Click Send Later icon or menu again
9. Select View > Character Coding  appropriate     character coding from menu. 
10. Click Send Later icon or menu ***		 * Confirm message is popped up no matter with fomat is text only or HTML

** Confirm message is popped up no matter with fomat.
                                                               ** Send Later Error message is popped up.

*** Confirm and Send Later error messages are no longer popped up

 
Offline07 For localized build only
Not downloaded Message		 1. Open Mail.
2. Go to File > Work Offline 
3. Click Not download button 
4. Try to open a message 		 Not downloaded message from server is displayed properly.
Offline08 Downloading  News folder 1. Open Mail.
2. Go to File > Offline > Download/Sync Now, click Select button.
3. Click checkbox news folders.
4. Click OK (then wait for synchronization to complete).
5. Go to File > Work Offline.
6. Try to access the messages in a folder you selected for offline usage.		 1. Non-ascii subjects are displayed properly.
2. Non-ascii characters in body of message are displayed properly.
3. Non-ascii sender is displayed properly.
