Feature Test Spec: Mail and News Component

Mail and News Component: Print Messages

Modified and maintained by Esther Goes

Description

Printing involves out-putting the displayed information on-to the printed media or electronic file. This test plan focuses on printing the content of the mail message to a device/file.Verify the expected results visually after each test and that all elements of the email messages has printed out correctly.

  1. Mail can be printed from various Mail/News windows
    1. 3 Pane UI
    2. 2 Pane UI (3 Pane UI collapsed to 2 pane UI)
      1. Collapse the Thread pane only and print a message
      2. Collapse the Folder pane only and print a message
    3. 1 Pane UI (3 Pane UI collapsed to 1 pane UI)
      1. Collapse the Thread pane and Folder pane then print a message
    4. Message Window
    5. Compose Window
  2. Ways to Print a Message
    1. Menu (File|Print)
    2. Toolbar (Print icon)
    3. Mnemonics (Ctrl+P)
    4. Context Menu
  3. Print Preview
    1. View Next page
    2. View Previous Page
    3. View Two Pages
    4. Zoom In
    5. Zoom Out
    6. Print
    7. Close/Cancel print preview
  4. Printer Properties (dependent on Print driver)
    1. Print Landscape
    2. Print Portrait
    3. Try a few other Printer options just to make sure app doesn't crash
  5. Number of messages that can be Printed
    1. Single message
    2. Multiple messages (Request for Enhancement)
    3. Selective messages (Request for Enhancement)
    4. All messages (Request for Enhancement)
  6. Mail message and news articles with the following contents:
    1. Text
    2. Graphics
    3. Java applets
    4. Vcards
    5. Signature

    7. Background Image/texture
    8. Other
  7. Print to different media
