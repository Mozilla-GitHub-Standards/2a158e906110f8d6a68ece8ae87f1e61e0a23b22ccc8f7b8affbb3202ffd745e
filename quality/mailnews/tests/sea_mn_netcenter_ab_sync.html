Feature Test Spec

6.0 Project
Mail and News Component
Netcenter Communications Services - Address Book Sync
(this is a defunct test plan since this feature is specific to Netscape builds only, nbaca 2002-05-07)

Written by: Peter Mock
3/05/2001 updated by Fenella Gor
2/09/2002 updated by Ninoschka Baca

References:
UI Specification: Revised Address Book Sync spec
Other: Address Book sync ideas from PRD

Description: Allows Netscape Mail users to store and retrieve address book information remotely as well as locally using synchronization between Netscape Mail's local address book and remote address book servers such as Netscape WebMail AB and AOL AB.

Criteria for synchronization:
1) users has a Netscape Webmail account
2) One screen name per profile can be used to sync
3) Synchronization can be initiated manually.

For simplicity, AB denotes Address Book. WM denotes WebMail.
To verify successful synchronization, there are 4 methods of verification to ensure data integrity of transfer between client and
a) synchronize to Netscape WebMail or AOL Screen Name
b) telnet to the server to verify values
c:) access AOL web site ab41.test.aol.com:5000.
Note: Method (b) and (c) require access
(d) checking the values on the remote address book server via the browser.

Goal: to ensure data integrity of data transfer between the client and server.

* This test spec does not cover synchronization between palm pilot since this version of Netscape Mail does not support directly in this version.


Visual
  1. Open the Address Book
    1. If you have a WM account, verify you have a sync button on the tool bar and that it is enabled
    2. If you have a WM account, verify you have a sync menu option under the File menu
    3. If you do not have a WM account verify the sync button is disabled
    4. If you do not have a WM account, verify you still have a sync menu but that it is disabled under the File menu
  2. Click on the Sync button for the first time.
    1. Verify that you receive a brief description explaing AB Sync
      1. Verify text description is correct
      2. Verify a secondary dialog appears asking for a User Name (aka Screen Name) and Password. This dialog is also known as the Password dialog
        1. Verify text in the second dialog is correct
  3. At the AB Sync password dialog
    1. For the first AB Sync
      1. Verify if you enter the wrong User Name (aka Screen Name) that you receive a warning message
      2. Verify if you enter the wrong password that you receive a warning message
      3. Verify if you enter the correct login and password, you receive feedback/progress information on the status bar.
      4. Verify you can cancel password dialog
    1. For additional AB Syncing
      1. Verify if you enter the wrong User Name (aka Screen Name) that you receive a warning message
      2. Verify if you enter the wrong password that you receive a warning message
      3. Verify if you enter the correct login and password, you receive feedback/progress information on the status bar.
      4. Verify you can cancel password dialog
Basic Functionality
  1. Verify you can cancel synchronization and the Seamonkey AB and Netcenter WM is left unchanged
  2. Create a Seamonkey Address Book card and verify synchronization of all the fields.
    1. Name tab information consists of name section, internet section, and phone section.
      1. first name
      2. last name
      3. display name
      4. nick name
      5. e-mail address
      6. secondary e-mail address
      7. send email as plain text (boolean true/false value)
      8. home
      9. work
      10. cell
      11. fax
      12. pager
      13. cellular
    2. Address tab information consists of home section and work section
      1. home address
      2. home additional address
      3. home city
      4. home state
      5. home zip
      6. home country
      7. home web url
      8. title
      9. department
      10. organization
      11. work address
      12. work additional address
      13. work city
      14. work state
      15. work zip
      16. work country
      17. work web url
    3. Instant Messenger tab
      1. information displayed once logged into AIM such as screen name
      2. information that can be edited such as buddy list
    4. Other tab
      1. custom field 1
      2. custom field 2
      3. custom field 3
      4. custom field 4
      5. notes
  3. Create a Seamonkey AB mailing list and verify synchronization of all the fields.
    1. mailing list name
    2. mailing list nickname
    3. mailing list description
  4. Create a WebMail card and verify synchronization of all the fields.
    1. Name tab information consist of name, internet, and other section.
      1. first name
      2. last name
      3. primary email
      4. secondary email
      5. web page
      6. home
      7. work
      8. cell
      9. fax
      10. pager
    2. Address tab information consist of primary mailing address, home, and work sections
      1. primary mailing address (h/w)
      2. home address
      3. home additional address 1
      4. home additional address 2
      5. home city
      6. home state
      7. home zip
      8. home country
      9. home web url
      10. title
      11. company
      12. work address
      13. work additional address 1
      14. work additional address 2
      15. work city
      16. work state
      17. work zip
      18. work country
      19. work web url
    3. Other tab information consist of personal user info
      1. screen name
      2. spouse name
      3. family name
      4. birthday
      5. anniversary
      6. notes
  5. Modify an existing replicated AB card one field at a time then synchronize the data.  Verify the changes has been or not been replicated to the Web Mail AB
    1. verify data integrity
      1. Modify first name
      2. Modify last name
      3. Modify email address
      4. Modify secondary email address
      5. Modify nickname
      6. Modify work address
      7. Modify home address
      8. Modify the AIM buddy list
      9. Modify information under other section
      10. synchronize a AB card that differ only by capitalization
    2. Data modified at Seamonkey AB stamped before data modified at WM. Synchronize after modification and verify that changes were replicated.
      1. Add AB card to Seamonkey AB
      2. Add an additional five cards to Seamonkey AB
      3. Import an existing AB.
    3. Data modified at Seamonkey AB stamped before data modified at WM. Synchronize after modification and verify that changes were not replicated.
      1. Delete one AB card from Seamonkey AB
      2. Delete two or more AB card from Seamonkey AB
      3. Delete one AB card from Web Mail
      4. Delete two or more AB card from Web Mail
      5. Delete one AB card from Seamonkey AB then add back the same card
      6. Delete one AB card from Web Mail then add back the same card
      7. Don't make any changes then synchronization again
    4. Data modified at Seamonkey AB stamped after data modified at WM
      1. Repeat the tests from previous section
    5. (rare but possible) Data modified at Seamonkey AB stamped at same time as data modified at WM
      1. Repeat the tests from previous section
  6. Modify an existing replicated Web Mail AB card one field at a time then synchronize the data.  Verify the changes has been replicated to the Communicator AB..
    1. repeat previous section except from Web Mail AB starting point.
  7. Synchronization of one Seamonkey AB mailing list. Verify the changes have been synchronize
    1. verify data integrity
      1. Modify list name
      2. Modify list nickname
      3. Modify description
    2. data modified at Seamonkey AB stamped before data modified at WM
      1. Add a name to the list
      2. Add several names to the list
      3. Delete a name from the list
      4. Delete then add back the same name from the list
      5. Don't make any changes - synchronization again to verify no data was transferred
    3. data modified at Seamonkey AB stamped after data modified at WM
      1. Repeat the tests from previous section
    4. (rare but possible) data modified at Seamonkey AB stamped at same time as data modified at WM
      1. Repeat the tests from previous section
AB Card Mapping
  1. Verify the AB card mapping of Seamonkey to WebMail
    1. field lengths of corresponding client and web mail AB matches
    2. fields that do not have a corresponding web mail should be allocated sufficient field length
  2. Verify the actual length of each Seamonkey AB card field.  Type as many character in to test for buffer overrun. (No information on field limitation.
  3. Export a synchronized Seamonkey Personal Address Book.  Import the ldif into a new address book.  Verify all information is imported.
Error Handling Situations
  1. Data Collision
    1. Synchronize an AB card that was previous synchronize
    2. Synchronization a list that already exist
    3. Synchronize an AB card that was deleted then added again
    4. Synchronize a new Seamonkey AB card that was added and same time as WebMail card.
  2. Data Loss
    1. Start a synchronization session and interrupt process.  Verify no data was lost
    2. Restart synchronization after an unsuccessful attempt.  Verify no AB record duplicated or missing
    3. Simulate system crash (terminate application).  Verify no data was corrupted.
    4. Rename the Messenger Personal Address Book then try to re-synchronization.
  3. Concurrent access of Netcenter WebMail AB file
    1. Log into the same Web Mail account from two different stations.  The Seamonkey address book should contain a number of existing cards (TBD - enough to force an access violation) and WebMail address book is empty.  Try to synch the data.
      1. Verify you can not access the same WebMail AB at the same time. (criteria: the Seamonkey AB files you're syncing must contain completely different data)
      2. Verify you receive the appropriate action from the WebMail server (warning, being logged off, etc.)
      3. Verify you do not lose data
    2. Repeat the previous test case except, have the client address books be empty and WebMail containing a certain number of existing cards (TDB).
    3. Repeat the previous test case except, have the each client address book and the WebMail containing a certain number of existing cards (TDB).
Stress Testing
  1. Try to synch the maximum number of characters in an AB field, exact number to vary in each field.
  2. Verify you receive a warning that you are nearing 1500 card
  3. Try to synch the maximum number of records allowed: 1500
