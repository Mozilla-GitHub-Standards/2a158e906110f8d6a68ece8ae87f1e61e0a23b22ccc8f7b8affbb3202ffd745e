Feature Test Spec: Mail and News Component

Mail and News Component: Search Messages: Basic Functionality within Mail

Description

General UI and news oriented searching are areas covered under separate test specs.

References

http://www.mozilla.org/mailnews/specs/

Each of the tests in this search testspec should be run against IMAP and POP mail servers.

As of 6.0, some of the displayed criteria options are not implemented or are incorrect for the server type. Many known bugs.

  1. Check search is generally functioning when search applied to
    1. All mail folders (server folder)
    2. Selected folder
    3. Reserved mail folder
  2. BASIC: Execute a simple single search for each criteria to verify it's engaging:
    1. a search which will yield at least one match
    2. a search which will yield multiple matches
    3. a search which will validly yield no matches
  3. Executing searches:
    1. Single criteria search : Note: the And/Or function shouldn't really enable until multiple criteria lines are entered for a search. This is a current bug and will likely stay that way for the next release.
      1. AND option selected (doesn't really make sense, but check search isn't affected by this)
      2. OR option selected (doesn't really make sense, but check search isn't affected by this)
    2. Multiple criteria
      1. AND
        • Sample each criteria type available
        • No result: matches on only one/some of the specified criteria
        • Search yields matches on all of the specified criteria
        • No match on either/any criteria
        • Illogical/negating AND criteria
        • two criteria search
        • three criteria search
        • four criteria search
        • five criteria search
      2. OR
        • Sample each criteria type available:
        • Search yields matches on one of the specified criteria
        • Search yields matches on all of the specified criteria
        • No match on either/any criteria
        • two criteria search
        • three criteria search
        • four criteria search
        • five criteria search
      3. Switching AND/OR: Verify ability to switch between logical AND ("match all of the following") and logical OR ("match any of the following"):
        • No allowance for combination AND OR searches.
        • Verify for newly entered criteria.
        • Verify on criteria already specified for AND switching to OR.
        • Verify on criteria already specified for OR switching to AND.
        • Verify this can toggle back and forth
    3. Tests for TEXT FIELD values:
      1. Partial string match
      2. Special characters
      3. Single word search
      4. Multiple word search
      5. Really short string
      6. Really long string
      7. cut and paste from other sources into text field :
        1. without carriage return
        2. with carriage return
        3. with special characters
    4. Tests for PRIORITY values:
      1. sample each priority value
      2. search headers which have had priority value changed by filter
    5. Tests for STATUS values: (Note: I don't believe this is implemented)
      1. sample for each status value
      2. search headers which have recently changed status (by filter, by actions like forwarding, replying, etc.)
    6. Tests for DATE selector values: (Note: No date selector widget available, just a text field -- bug #90289/92174)
      1. current
      2. past
        • month
        • date
        • year
        • within the folder's message range
        • outside the folder's message range
      3. future
        • month
        • date
        • year
        • within the folder's message range
        • outside the folder's message range
    7. Tests for AGE IN DAYS values:
      1. 0
      2. small range - single digit
      3. medium range - double digit
      4. longer range - over double digit:
    8. Stop search
    9. Consecutive searches:
      1. Clear in between
      2. Don't Clear in between
    10. Close dialog while search ongoing
  4. Results Pane
    1. Sorting order after a search
      1. Subject
      2. Sender
      3. Date
      4. Priority
      5. Location
    2. Delete Message
      1. Use "Delete" button:
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection across different folders
      2. Use the delete key on the keyboard: (Not implemented)
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection across different folders
      3. Delete via context menu: (Note: not implemented in 6.0 and won't be for next release - bug 66970)
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection across different folders
    3. File Message
      1. Use "File" button:
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection across different folders
        • IMAP--> IMAP (single imap account)
          • online
          • offline
        • IMAP --> local mail
          • online
          • offline
        • local mail --> IMAP (single account)
          • online
          • offline
        • single POP
          • online
          • offline
      2. Use file context menu item (Note: not implemented in 6.0 and won't be for next release - bug 66970)
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection across different folders
      3. Use drag and drop (Note: not implemented in 6.0, won't be for next release - bug 66955)
        • for single result selection
        • for contiguous multiple result selection
        • for non-contiguous multiple result selection
        • for multiple result selection located in different folders
        • IMAP--> IMAP (single imap account)
          • online
          • offline
        • IMAP --> local mail
          • online
          • offline
        • local mail --> IMAP (single account)
          • online
          • offline
        • single POP
          • online
          • offline
    4. Opening a message to a Message Window
      1. Select message and press enter
      2. Double click on message
      3. Use Open button (Note: not implemented in 6.0, is implemented for next release)
    5. Select a message and click on "Go to Message Folder" button, make sure context for message is correct:
      1. message folder in various sort methods
      2. use IMAP selection
        • online
        • offline
      3. use local mail selection
        • online
        • offline
      4. use POP (single account) selection
        • online
        • offline
  5. Error or adverse conditions
    1. No matches found
    2. IMAP server is inaccessible
    3. No such mail folder (folder deleted from another machine)
    4. Connection interrupted while search in progress
    5. Offline search for IMAP mail not implemented. Probably should error.
