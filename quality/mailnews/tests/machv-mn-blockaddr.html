Feature Testspec

        Project:  MachV/Mozilla
        Mail and News component
        Block Address

12/18/01  Notice:  This feature has been placed on the cut list.  Ref: bugzilla bug #71413

Written by: Laurel Carlson

Last Modified :

 12/03/01 draft 

References:

Prefill UI spec:   http://www.mozilla.org/mailnews/specs/filters/FilterPrefill1.html#b
Main Filters Messages UI Spec: http://www.mozilla.org/mailnews/specs/filters/

  1. General UI components :
    1. Menu item Message|Block Address
      1. Should be disabled when in Local Folders (server level, folder or message)
      2. Should be disabled when in News accounts (server level, folder or message)
      3. Should be disabled for multiple thread pane selection
      4. Enabled in thread pane
      5. Enabled in message pane
      6. Enabled in standalone message
    2. Envelope header popup menu:
      1. Should be disabled in Local Folders
      2. Should be disabled in News accounts
      3. Enabled in 3-pane mail header envelope
      4. Enabled in standalone window header envelope
    3. No message context menu (thread pane, message pane, standalone)?
    4. Access via Keyboard Shortcut?
    5. Confirmation dialog when using Block Address menu/popup/context item:
      1. Descriptive text (check spec in case of updated text):  "New messages from the email address <name@account.com> will automatically be moved to the Junk Mail Folder."
      2. Checkbox "Do not show this dialog again"
        • default OFF
        • state interacts with confirmation checkbox in Blockd Addresses list
      3. General OK/Cancel
    6. Blocked Addresses List 
      1. Accessible from main menu item Edit|Blocked Email Addresses
      2. TBD: Maybe accessible from Tools
      3. Text field 
      4. List box
      5. Add/Remove buttons
      6. Option checkbox/edit field for when to empty Junk Mail Folder 
        • default ON
        • default value 30 days
        • edit box error checking: non numeric, extreme range of days?
      7. Option checkbox for confirmation when blocking address
        • default ON
        • coincides state with confirmation dialog checkbox
      8. General OK/Cancel
      9. Tabbing  access and order
      10. Mnemonics
    7. Junk Mail Folder
      1. Is under Local Folders
      2. Created by default for new user
      3. Created on the fly for migrated older profile?
      4. Email message present in Junk Mail folder explaining Block Sender feature and purpose of the Junk Mail Folder
    8. Prefs:  Any prefs settings for the feature?
  2. Using Block Address via main menu item
    1. Launches the confirmation dialog: fills in appropriate (From) email address in confirmation
    2. Confirmation can be cancelled:
      1. name not added to blocked list (must access list separately to check that)
      2. confirmation dialog dismissed
      3. user returned to mail selection:
        •  main mail window
        • standalone message window
    3. Upon confirmation:
      1. name  (From) added to blocked address list (but the list ui doesn't display, must access the list separately)
      2. confirmation dialog goes away
      3. user returned to mail selection:
      4.  main mail window
      5. standalone message window
    4. Available and operable message selected in thread pane:
      1. message pane open or closed
      2. available with multiple selection? try it.
    5. Check error handling for proper header/email format?
    6. Check error handling for duplicate addresses?
    7. Filter maintained  after exit:
      1. check blocked address list UI
      2. check file on disk (what filename?)
      3. check filter still works next session
  3. Block Address via header popup
    1. Launches the confirmation dialog: fills in appropriate (From) email address in confirmation
    2. Confirmation can be cancelled:
      1. name not added to blocked list (must access list separately to check that)
      2. confirmation dialog dismissed
      3. user returned to mail selection:
        •  main mail window
        • standalone message window
    3. Upon confirmation:
      1. name  (From) added to blocked address list (but the list ui doesn't display, must access the list separately)
      2. confirmation dialog goes away
      3. user returned to mail selection:
        •  main mail window
        • standalone message window
    4. Check error handling for proper header/email format?
    5. Check error handling for duplicate addresses?
    6. Filter maintained  after exit:
      1. check blocked address list UI
      2. check file on disk (what filename?)
      3. check filter still works next session
  4. Using the Blocked Addresses list (Add/Remove/View)
    1. Add address manually
    2. After add, make sure incoming message matching address filtered to Junk Mail
    3. Add multiple addresses: scroll appears appropriately
    4. Add long address
    5. Check error handling for proper header/email format?
    6. Check error handling for duplicate addresses?
    7. Recuperates smoothly from error on input
    8. Addresses listed in alphabetical order
    9. Delete address manually:
      1. from top of list
      2. from middle of list
      3. from bottom of list
      4. list should resort and redraw properly
    10. Delete multiple addresses:
      1. from top of list
      2. from middle of list
      3. from bottom of list
      4. list should resort and redraw properly
    11. Add a name via menu or header popup while the blocked address list is open
    12. Delete a name which was just added via automatic (menu, header popup) process
    13. Changes to list are preserved through exit:
      1. check blocked address list UI
      2. check file on disk (what filename?)
      3. check filter still works next session
      4. Will blocked address list be per account or global for profile?
  5. Running filters against blocked address:
    1. Messages filtered to Junk Mail which match a manually added blocked address
    2. Messages filtered to Junk Mail which match an automatically added blocked address
    3. Messages stop being filtered to Junk Mail for addresses which were removed from blocked address list
    4. TBD/Open Issue:  Messages filtered After-the-Fact on confirmation when Block Address command used
    5. TBD/Open Issue:  Use manual After-the-Fact on blocked address list?
    6. Blocked address list to take precedent over regular filters (I assume so).
      1. Create various filters then send from a blocked address which would meet the filter criteria.  Result should be that the message is moved to Junk Mail and no further filter action done for the message.
      2. Create a prefilled Sender filter on a blocked address.   Send from that sender, should be moved to Junk Mail without further filter action.
    7. Any checking on the part of Prefill or Block Address to warn user of existing filter or blocked address on same address? (Probably not.)
    8. Any logging for Block Address firing?
    9. Make sure once a blocked address is deleted:
      1. New messages from the previously blocked sender matching no other filters are not filed to Junk Mail; they should be received in Inbox normally.
      2. New messages from the previously blocked sender which are involved in other filters are appropriately acted upon.
      3. Existing messages from the previously blocked sender may be filtered using manual filter execution ("After-the-Fact")
  6. The Junk Mail folder:
    1. Presence of Junk Mail Folder:
      1. New profiles: Junk Mail created by Default under Local Folders
      2. New accounts in older profile: Junk Mail created under Local Folders
      3. Older profile:  Junk Mail created when blocked address feature used (at filter time or at creation of blocked address?)
      4. Migrated:  Assume Junk mail created on the fly when feature used
      5. Shouldn't be able to be deleted or renamed?
      6. Will it recreate if  somehow deleted?
      7. Shared by all  mail accounts
    2. Can manually delete messages in junk folder
      1. single selections
      2. multiple selections
    3. Can generally use folder ui basics:
      1. scroll
      2. read messages
      3. Sort
      4. Menu items -- are any restricted, disabled? (Like Views)
      5. Create subfolders of Junk Mail
    4. Aging/auto-delete mail in Junk Mail folder
      1. Pref/option is located in Blocked Address List
      2. When will the deletions take place -- on exit, on start-up???
      3. Default works to empty messages older than 30 days
      4. Change value from the default, works to empty messages older than N:
        • recent messages, single digit
        • older messages, double digit - triple digit
      5. Disable aging, make sure no mail messages aged/deleted
