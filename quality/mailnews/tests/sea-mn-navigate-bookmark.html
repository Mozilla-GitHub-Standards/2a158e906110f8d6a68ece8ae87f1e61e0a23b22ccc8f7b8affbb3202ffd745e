Feature Testspec

Project: 5.0 (Seamonkey)
Mail and News component
Navigate: Bookmarks

Original document was written by: Laurel Carlson for 4.5 (Nova)
Modified and maintained by: Fenella Gor for 5.0 (Seamonkey)

References:
5.0 Seamonkey UI Specification:
Test case management system: n/a
Number of test cases: approx. 45

This feature is NOT available in Netscape 6.0

The purpose of these tests is to ensure that you can add bookmark, edit bookmark and access bookmark for mail and news messages.

  1. Access bookmark mail/news message: set focus on each of the following window or message panes, add bookmark via Communicator|Bookmark and/or CTL-D(Windows), verify that a mail/news message is added to Bookmark:
    1. Windows to be tested:
      1. 3 pane UI window (Folder pane, thread pane and message pane open)
      2. 2 pane UI window (Folder pane collapsed)
      3. 2 pane UI window (Message pane collapsed)
      4. 1 pane window (Folder pane and Message pane collapsed)
      1. Message window
      2. standalone Message Center window
  2. Basic tests for bookmark for mail/news messages
    1. 3 pane window
      1. Add Bookmark (from Communicator|Bookmarks dropdown menu)
        1. verify that the bookmark message name is correct as compared to its original mail/news message.
      2. Edit Bookmark (from the Communicator|Edit Bookmark and/or CTRL-B (window), selecting Edit Bookmark)
        1. Name: verify bookmark message name is correctly displayed
        2. url: verify url links works (the page should display in the browser.)
          • Name: verify bookmark message name is correctly displayed
          • url: verify url links works (the page should display in the browser.)
          • Verify delete bookmark
          • Verify close bookmark works
      3. After you added bookmark, exit Browser and then bring up the Browser again.
        1. Verify that mail/news bookmarked message exists and that you can access the mail/news bookmark message from Bookmark:
      1. Repeat the above test cases in a 2 pane UI window (Folder pane collapsed)
      2. Repeat the above test cases in a 2 pane UI window (Message pane collapsed)
      3. Repeat the above test cases in a 1 pane window (Folder pane and Message pane collapsed)
      4. Repeat the above test cases in a Message window
      5. Repeat the above test cases in a standalone Message Center window
  3. Variations of mail/news messages to bookmark ( refer to Other bookmark Conditions for online and offline testings):
    1. Messages on News servers: bookmark a message from each of the following news server, verify Add bookmark works. Check Edit bookmark to ensure that the bookmarked message is displayed correctly and that you can access the message from bookmark.
      1. Netscape 2.0
        • secure
        • not secure
      2. Netscape 3.0
        • secure
        • not secure
      3. Netscape newest (4.0)
        • secure
        • not secure
      4. Password protected group: Add bookmark from a news group that has password protection and edit bookmark to ensure bookmark message is correct and that you can access the message using bookmark.
    2. Messages on Mail servers: bookmark a message from each of the following mail server, verify Add bookmark works. Check Edit bookmark to ensure bookmark messags are displayed correctly and that you can access the message using bookmark.
      1. IMAP
        • nsmail-2
        • nsmail-5
      2. POP3
        1. local mail
  4. Other bookmark conditions:
    1. Follow bookmark when offline
      1. Mail
        1. IMAP
          • Add bookmark on message that is offline (message already downloaded), edit it and access it
          • Add bookmark on message that is offline|synchronize(message not yet downloaded, but synchronize will download it), edit bookmark and access bookmark.
        2. POP3
          • Add bookmark on message that is offline, edit it and access it
          • Add bookmark on message that is offline|synchronize, edit bookmark and access bookmark.
      2. News
        1. Secured server
          • Add bookmark on message that is offline (message already downloaded), edit it and access it
          • Add bookmark on message that is offline|synchronize(message not yet downloaded, but synchronize will download it), edit bookmark and access bookmark.
        2. Non-secured server
          • Add bookmark on message that is offline, edit it and access it
          • Add bookmark on message that is offline|synchronize, edit bookmark and access bookmark.
    2. Bookmark message is in local database (on-line)
      1. Mail
        1. IMAP:
          • Use the IMAP server, click on a mail message andl download the message to cache. Bookmark it and check if the mail bookmarked message is correct and that you can access the mail message from Bookmark
        2. POP3:
          • Use the POP3 server, click on a mail message andl download the message to cache. Add the mail to Bookmark, do an Edit bookmark and verify that you can access the mail message using Bookmark.
      2. News
        1. secured server (eg poisonivy:563):
          • clicking on a mail messagewill download the message to cache. Bookmark it and check if the news bookmarked message is correct and that you can access the news message from Bookmark
        2. non-secure server (eg: poisonivy):
          • verify add bookmark, and edit bookmark and that you can access the message using Bookmark.
    3. message not downloaded locally (0n-line but close the message pane so that it will not download message locally)
      1. Mail
        1. IMAP
          • clicking on a mail message will not download the message to cache because message pane is closed. Bookmark the mail message and check if the bookmarked mail message is correct and that you can access the message from Bookmark
        2. POP3
          • Not applicable because message is from the local disk.
      2. News
        1. Secured server (eg: zia:563)
          • Do a search message and bookmark a message and edit bookmark to ensure that the bookmarked news message name is correct and that you can access the news message from Bookmark.
        2. Non-secured server (eg: zeo)
          • Do a search message and bookmark a news message. Do a Edit bookmark to ensure that the bookmarked news message name is correct and that you can access the news message from Bookmark.
    4. Bookmark management (Will find out if browser group covers this section ??)
      1. Create a new folder
      2. File mail/news message bookmarks into the new folder
      3. Re-organize the bookmarked mail/news messages
