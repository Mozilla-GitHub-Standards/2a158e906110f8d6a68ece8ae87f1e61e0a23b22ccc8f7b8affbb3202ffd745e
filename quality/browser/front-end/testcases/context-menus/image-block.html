Test Case Description for: Block Images from this Server

Author: Sarah Liberman

Purpose:

Test the Block Images from this Server item in the context menu.

Initial Conditions:

If you are using the commercial version, Netscape 6, you'll need to do the following since this feature is off by default. (It is on by default in the Mozilla version.)

  1. Quit the application.
  2. Add the following line to your prefs.js file.

    3. user_pref("imageblocker.enabled", true);

  3. Restart the application. For the sake of simplicity, make sure you have only one browser window open.

Note. Bring up the context menu by using the right-mouse button (Unix and Win32), or Command+click (Mac OS).

Steps/Description:
  1. Go to http://bugzilla.mozilla.org .
  2. Bring up the context menu over the banner at the top of the page.
  3. Select Block Image from this Server from the context menu.

Expected Results:     

Reload the page and verify that the banner (which is from the http://www.mozilla.org server) no longer displays --it should be just a black block now. However, the picture of the ant should still be visible, as it comes from another server, http://bugzilla.mozilla.org .

Note. To restore seeing the banner, simply select Tasks > Privacy and Security > Image Manager > Allow Images from this Site. You'll then need to reload the page to see the image again.
