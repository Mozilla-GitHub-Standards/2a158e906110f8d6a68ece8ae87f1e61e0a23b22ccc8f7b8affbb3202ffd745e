Test Case Description for: Reload and Reload Frame menu items

Author: Sarah Liberman

Purpose:

Test the Reload and Reload Frame items in the context menu.

Initial Conditions:

Start the Mozilla browser (Navigator). For the sake of simplicity, make sure you have only one browser window open.

Note. Bring up the context menu by using the right-mouse button (Unix and Win32), or Command+click (Mac OS).

Test #1
Steps/Description:
  1. Go to a page without frames where the content is frequently updated/changed dynamically, such as http://tinderbox.mozilla.org/showbuilds.cgi?tree=SeaMonkey
  2. Take note of the time underneath the "tinderbox" title.
  3. Wait about 5 minutes.
  4. Bring up the context menu, and select Reload.

Expected Results:

The current page should be reloaded, but the time underneath the "tinderbox" title should be 5 minutes later than in step 2. If the tree is busy, you might even notice that the checkin table might have changed with the latest "Build Time" and/or "Guilty" info.

Test #2
Steps/Description:
  1. Go to a page with frames where the content is frequently updated/changed dynamically, such as reload-frames.html.
  2. Take note of the time underneath the "tinderbox" title in the top frame.
  3. Wait about 5 minutes.
  4. Click within the top frame to make it the active frame.
  5. Bring up the context menu, and select Reload Frame.

Expected Results:

The top frame should be reloaded, but the time underneath the "tinderbox" title should be 5 minutes later than in step 2. If the tree is busy, you might even notice that the checkin table might have changed with the latest "Build Time" and/or "Guilty" info.

However, the bottom frame (pointing at http://slashdot.org) should remain unchanged and not be reloaded.
