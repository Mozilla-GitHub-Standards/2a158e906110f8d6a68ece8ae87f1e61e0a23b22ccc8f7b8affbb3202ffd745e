Test Case Description for: Simple context menus

Author: Sarah Liberman

Purpose:

Test simple context menus for non-frame and frame pages, as well ftp listings where nothing has been selected.

Initial Conditions:

Start the Mozilla browser (Navigator). For the sake of simplicity, make sure you have only one browser window open.

Note. Bring up the context menu by using the right-mouse button (Unix and Win32), or Command+click (Mac OS).

Test #1
Steps/Description:
  1. Go to a page that doesn't have any frames, eg, http://www.mozilla.org.
  2. Bring up the context menu over a region of the content area which does not contain a link, image or any form control. Whitespace or text would work, such as near the top of the second column, "Mozilla News".

Expected Results:

You should see the following type of context menu. Style might vary depending on the theme you're using, but the content, spelling and order of the items should be very similar.

Test #2
Steps/Description:
  1. Go to an ftp listing, eg, ftp://ftp.mozilla.org/pub/.
  2. Bring up the context menu over a region of the content area which does not contain a link.

Expected Results:

You should see the same type of context menu as seen in Test #1 above.


Test #3
Steps/Description:
  1. Go to a page that does have frames, eg, simple-frames.html.
  2. Bring up the context menu over a region of the content area which does not contain a link, image or any form controls.

Expected Results:

You should see the following type of context menu. Style might vary depending on the theme you're using, but the content, spelling and order of the items should be very similar.
