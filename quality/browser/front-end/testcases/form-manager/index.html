Test Case Matrix for Form Manager

Written by: Terri Preston

Please note that this is a work-in-progress and is maturing as the feature matures. If there is a description and no test case link, just try to perform the actions described.

Couldn't find a particular test case here? I encourage you to send additions, corrections and suggestions! In addition, feel free to check out the corresponding Form Manager Testplan

The Form Manager is also known as Autofill.

Since the forms from the Demonstration menu item are just sample forms, none of the data entered into them are sent to a server. That's why the fields are cleared after you click the appropriate form submit button.

New Feature Tests
(these tests are currently inactive, please do not write bugs on these testcases)
Feature/Function Description
Form Manager - Autocomplete 113893 Implement New Form Functionality (waiting on bug status)
Form Manager - Pop-up Verify that a form fill popup menu is activated when a field gets accessed that has been saved in form manager data.
Form Manager - Permission Zones * Longer range: there is discussion about introducing Internet permission zones. Some of the settings shown here may be controlled by such zones (Waiting for Implementation)
Active test cases for Seamonkey
Feature/Function Description
Save Form Data - edit menu Verify that you can save a form's data from the edit menu.
Save Form Data - right click Verify that you can save a form's data by right clicking on the form itself.
Save Form Data - dialog yes Verify that you can save a form's data from the popup dialog box and you click yes.
Save Form Data - dialog no Verify that you can stop a form's data from being saved from the popup dialog box, and you click no.
Save Form Data - dialog never Verify that you can stop a site's data from being saved from the popup dialog box, and you click never for this site.
Save Form Data - never return Verify that you can save a form's data from the popup dialog box, never for this site and you are never asked again upon returning to the site.
Prefill - OK Dialog Verify that when you encounter a Web site that could be prefilled, that you are given the pre-fill dialog and when you click ok, the information is filled into the form.
Prefill - cancel dialog Verify that when you encounter a Web site that could be prefilled, that you are given the pre-fill dialog and when you click cancel, the information is not filled into the form.
Prefill - choose Verify that when you encounter a Web site that could be prefilled and have several choices for fields that you are given the pre-fill dialog and can select from the appropriate values and the information is filled into the form.
Edit Form Data - new value for field Verify that if you add a new value for a field in Form Manager it is later offered as an option when prefilling a form using this field.
Edit Form Data - cancel Verify that if you enter data in the Form Manager and then click cancel, the data has not been saved
Delete Form Data - remove field value Verify that if you delete data in the Form Manager, that this data is not shown as an option when prefilling a form.
Delete Form Data - cancel Verify that if you delete data in the Form Manager then click cancel, the data has not been deleted
