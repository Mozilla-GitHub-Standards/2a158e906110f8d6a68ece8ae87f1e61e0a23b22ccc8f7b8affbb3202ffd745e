Test Case Matrix for Wallet: Password & Form Managers

Maintained by: Sarah Liberman

If a link doesn't exist under Test Case Links for a given test, just do the action described under its corresponding Description.

  • Password Manager tests
  • Form Manager tests

Couldn't find a particular test case here? I encourage you to send additions, corrections and suggestions! In addition, feel free to check out the corresponding test plan for Password and Form Managers.

Password Manager

The Password Manager is also known as Single Signon. btw, The phrase "signon information" refers inclusively to username, password and site information saved within the Password Manager.

Feature/Function/Test Case Description Test Case Links
Basic access & UI The main menu items and dialogs for this feature.  
Saving a password

Tests for saving signon information, including:

  • first-time use
  • test signon prefill
  • cancel prefill
  • changes in password for the same username
  • web page forms, as well as HTTP authentication
  
Saving multiple passwords

Tests for saving multiple signon information, including:

  • testing signon prefill
  • changes in password for the same username
  • web page forms, as well as HTTP authentication
  
Never saving passwords

Tests for never saving signon information for a given website; ie, site-dependent. Includes tests for both web page forms and HTTP authentication.

  
Not saving password Tests for simply not saving signon information at all; ie, site- independent. Includes tests for both web page forms and HTTP authentication.  
Delete password Tests for removing signon information. Includes tests for both web page forms and HTTP authentication.  
Delete all password and form data Tests removing both signon and form information that had been saved.

Form Manager

The Form Manager is also known as Autofill.

Note. Since the forms from the Demonstration menu item are just sample forms, none of the data entered into them are sent to a server. That's why the fields are cleared after you click the appropriate form submit button.

Feature/Function/Test Case Description Test Case Links
Basic access & UI The main menu items and dialogs for this feature.  
Active form capture

Capturing form data by actively selecting the Save Form Data command. Includes tests for:

  • Save Form Data from Edit menu
  • Save Form Data from context (popup) menu
  • Save Form Data from Tasks > Privacy and Security > Form Manager
  
Passive form capture Capturing form data by clicking a form's submit button.  
Cancel form capture Canceling data capture from either active or passive means.  
Prefill a form (simple)

Prefill a form on a web page with only one value per field. Includes tests for:

  • Prefill a form with the Prefill Form Data (preview) dialog
  • Prefill a form without the preview dialog
  • Prefill Form from Edit menu
  • Prefill Form from context (popup) menu
  • Prefill Form from Tasks > Privacy and Security > Form Manager
  
Cancel form prefill Cancel form prefill from the preview dialog.  
Edit form data

Tests for adding items in the Stored Form Data dialog. Includes tests for:

  • Add a new form field
  • Add a new value for a field
  • Add a new synonym for a field value
  • Cancel any changes made within the dialog
  
Delete form data

Tests for removing items in the Stored Form Data dialog. Includes tests for:

  • Remove a new form field
  • Remove a new value for a field
  • Remove a new synonym for a field value
  • Cancel any changes made within the dialog
  
Prefill a form (multiple field values) Prefill a form on a web page with multiple values per field.  
Prefill a form (multiple value synonyms) Prefill a form on a web page with multiple synonyms per field value.
