Accumulated performance results for the Preferences dialog

Date: 20 June 2002
By: Sarah E.V. Liberman

NEWS. As of April 2002, these tests are now run every other week (biweekly).

Tests:
  1. Time opening of Preferences dialog.
  2. Time panel switching from Navigator to Mail and Newsgroups.
Browsers tested:
  1. Netscape 6.x (commercial)
  2. Netscape 6.2.1 RTM bits
  3. Netscape 6.1 RTM bits This was remeasured because the performance machines were wiped clean on 20 December 2001.
  4. Netscape 6.0.1 RTM bits This was remeasured because the performance machines were wiped clean on 20 December 2001.
  5. Communicator 4.76
Machines used in the lab:
  • Linux:
    • Before 8 May 2002: Redhat 6.2, 128M RAM, 266 MHz Pentium II
    • As of 8 May 2002: Redhat 7.3, 128M RAM, 266 MHz Pentium II
  • MS-Windows: 98 Second Edition, 128M RAM, 266 MHz Pentium II
  • Macintosh OS 9.x:
    • Before 20 Dec 2001: Mac OS 9.0, 128M RAM, 266 MHz G3
    • After 20 Dec 2001: Mac OS 9.2.2, 128M RAM, 266 MHz G3
  • Macintosh OS X:
    • Before 8 May 2002: Mac OS 10.1.2, 128M RAM, 266 MHz G3
    • As of 8 May 2002: Mac OS 10.1.4, 128M RAM, 266 MHz G3
    • New. As of 12 June 2002: Mac OS 10.1.5, 128M RAM, 266 MHz G3
Notes:
  • Classic theme used. Sidebar off. XUL cache on, which is default for trunk, and also, I presume the default for 6.01 and later RTM builds.
  • Timings are in seconds and were done manually with a stopwatch. Reported values are the average of usually 3 takes.
    • Each Open Preferences measurement was taken during a unique session.
    • Panel switching measurements, on the other hand, were taken within the same session. (Didn't notice much differences btwn unique and same session measurements here.)
  • Timings that averaged under half a second are considered "instantaneous", imo. :)
  • New [non-migrated] profiles were created for each browser (three builds) on each platform (three).
