Performance results for the Preferences dialog

Date: 22 February 2001
By: Sarah E.V. Liberman

Machines used [in my cube]:
  • Linux: Redhat 6.2, 256M RAM, 500MHz Pentium-III
  • MS-Windows: NT 4.0, 128M RAM, 500MHz Pentium-III
  • Macintosh: Mac OS 9.0, 128M RAM, 450 MHz G3
Tests:
  1. Time opening of Preferences dialog. Each timing was done during a unique session.
  2. Time panel switching from Navigator to Mail and Newsgroups. All timings were done during the same session [I found no significant difference between timings done during the same or unique session anyhow].
Browsers tested:
  1. Communicator [Navigator] 4.76
  2. Netscape 6.01 [RTM bits]
  3. Netscape 6.5 [commercial] 2001.02.22.08 [Linux, Mac] and 2001.02.22.10 [Win32] bits
Notes:
  • After discussion with mcafee, we've decided that we reduce the panel switching tests from three to a single one --since the timings are very similar.
  • Classic theme used. XUL cache on [default for trunk, and I presume the default for 6.01].
  • Timings are in seconds and were done manually with a stopwatch. Reported values are the average of usually 3 takes.
  • Timings that averaged under half a second are considered "instantaneous", imo. :)
  • New [non-migrated] profiles were created for each browser [three] on each platform [three].
