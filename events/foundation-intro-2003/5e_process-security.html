  • Mozilla is built on a strong security architecture and default settings.
    • sandbox model for protecting code execution
    • browser and mail are independent of the operating system
    • support for activeX, JavaScript, other executable code in mail messages turned off by default
  • Audits of the entire source code baseline are conducted to identify problems
  • Extensive peer review reduces the chances of introducing new vulnerabilities
  • Academics and world renowed security experts study the source code independently and help the project to identify and reduce the chances of vulerabilities
  • Security@mozilla.org mail alias set up to receive incoming reports of potencial vulerabilities and allow our security team to evaluate and fix potencial problems.
  • Any discovered vulerabilities are fixed quickly and made available for deployment.
  • Openess and transperancy often lead to trust, and we believe help Mozilla to provide the most secure browser, e-mail, and internet application platform.
