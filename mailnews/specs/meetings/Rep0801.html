Netscape 6.1/Mozilla 0.9.4 Usability Summary Report

August 13, 2001

A usability evaluation was conducted with the Netscape 6.1 client on August 8 & 9, 2001. Issues that are relevant to Mozilla, as well as the commercial client, are summarized here.

A total of six users participated, in the evaluation. User experience ranged from intermediate to advance. Overall, participants were able to effectively use the products once they had located them. It was difficult in some cases for the people to find the various products with the menus or from icons that are provided in the interface. Also, there was a significant amount of confusion regarding the difference between the client based products and the web based counter parts that are provided as part of the Netscape website.

The main issues uncovered during testing are summarized below.

Problematical Tasks

  1. Read email and reply to messages
    • Users 1, 2, 3, and 6 all had to be prompted to access the client email program. These users all chose to access Webmail when asked to read email messages. After accessing Webmail, users were prompted to look for a way to check the other email address that they had been given. None of these four users were able to access the client-based email without help from the test facilitator.
    • User 4 began each new task by closing the active program and starting the new tasks from the program group. This user started the mail client from the program group.
    • User 5 was the only person in the evaluation that understood the difference between client-based mail and Webmail.
  2. Add sender to address book
    • User 1 was very confused about the difference between the collected address book and the personal address book. The user attempted to copy and address from one address book to the other but never succeeded in doing so.
    • User 2 was unable to locate the address book and had to be prompted to the address book.
    • Users were mostly successful with this task, however only one user in test clicked on the address in email message to add it to the address book. The other people wrote down the email address and made a new entry in the address book
  3. Use imported book mark
    • All of the users were able to quickly access the list of bookmarks. However users 2 and 6 noticed the personal bookmarks folder was empty and assumed that no bookmarks were imported from the previous version of Netscape.
  4. Add address to address book from card
    • All of the users were able to successfully access the address book with some difficulty finding it in the menus or via pull down menus. However, 4 of 6 users edited an existing entry rather than creating a new entry. This was due to some confusion about how to initiate a new address book entry.

 

Key Findings

Each usability concern is given a severity rating of 1 to 5 weighted by the issueâs expected frequency of occurrence and impact to the customerâs work.

  • 5Ê-  Cosmetic or detail defects that are irritating to customers. These errors can accumulate and give the product and ãunfinishedä or ãbuggyä impression to customers.

  • 4 - Defect causes a distraction to the customer but no real loss in the ability to get needed tasks accomplished.

  • 3 - Defect causes customer to change work strategy to get around problems with the interface. Work goals can still be accomplished with some effort.

  • 2 - Defect is a severe inhibitor to accomplishing work goals. In some cases, customers cannot successfully complete the task.

  • 1 - Defect is critical and results in system or data loss. Customer is unable to achieve needed work goals.
