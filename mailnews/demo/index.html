RDF WebMail Demo

Disclaimer

This is a technology demonstration of an RDF-based mail reader. It is intended to give developers a flavor of what is possible with these new technologies in the Mozilla Web browser. We make absolutely NO warranty about the behavior of this software. Do NOT use this software on live mailboxes for anything other than testing purposes.

How it Works

Mozilla contains two new interesting HTML elements. One is an RDF tree control (Shack), the other is an HTML compose form element (Ender). Using these widgets, plus HTML and JavaScript we can build a mail viewer that has many of the attributes of a classic "fat" mail client and yet preserves the flexibility and location independence of an HTML mail client. In this demo, a gateway converts between the IMAP/POP protocols and RDF representations of folder lists and message lists, as well as generating HTML representations of the overall UI. Individual messages are displayed as their native message type. In order to display your mailbox, your browser requests a specific URL from the gateway which uses IMAP or POP to get that data from your mailbox.

Security

This demonstration requires an e-mail id and a password in order to access your mail server. This password is transmitted in the clear between your web client, our server and then your mail server. Even worse, any messages you read are also transmitted in the clear, from your mail server to the web server and from the web server to your browser. As we said, this is only a technology demonstration. If this weak security is unacceptable to you please do not run the demo. Certainly do not use it with any accounts and passwords that have great value to you.

Performance

We know this demonstration is relatively slow. This is because the gateway currently doesn't remember any state while you are reading mail. In essence it has to login to the mail server for each message. We know this is silly and we are working on a faster gateway.

Feedback

This is just a demo of what is possible with RDF and the Shack widget. There are many directions to take this work, only one of which is to make it a fully fledged conventional mail reader.

Please send us your feedback at smartmail@mozilla.org.
