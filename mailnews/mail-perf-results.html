Mail Performance Results for Windows
Tested and compiled by Stephen Donner <stephend@netscape.com>

Machine Configuration:
  • Windows XP, Pentium 4, 1.7 GHz, 256MB of RAM.
  • Any area with '-' means that the area is impossible to test with only a stopwatch.
  • Areas with <0.30 reflect the fact that human reaction time is ~ 20ms, so these are general measurements.
Area Tested Methodology for the Area
Folder Loading Speed Time to load the summary file with 1,000 messages after launching the mail client (local folders). (When you save this, take the .txt extension off of the file.)
Thread Pane Scrolling Speed Time to scroll 1,000 messages in real-time from top to bottom. 
Message Display Speed Display 5 10kb messages from a POP inbox.
Thread Pane Sorting 1,000 message folder, sort by subject, sort by sender, and then sort by date. (Initial sort of date was done to inititiate sort state. Not timed). 
Message Reply Speed
 Click Reply All to a 2kb HTML message sent to 5 recipients.
Forwarding 30kb inline Forward a 30kb message inline.
New Window Time Click New Msg.
Batch Delete 5 Messages Select 5 messages at once and Delete (local folder). 
Mail Module Loading Time Launch the application by doing <[appname.exe] -mail> (if it supports/requires arguments to the executable)
