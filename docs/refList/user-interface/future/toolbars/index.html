Toolbars

Authored by: Irene Au
Date created: 4 March 1998

Summary

    This document describes possibilities for future toolbar work in Communicator.

Customizable toolbars

    What buttons go on the toolbar has always been a holy war. It would be great if toolbars were completely customizable so advanced users would be able to put more functionality on the toolbar if desired. However, several software design groups have data that few users actually do customize their toolbars; therefore, this feature never made it on the "in" list in the past.
    Alternatively, we can provide different modes of use. For instance, by default, we ship with a basic set of commands on the toolbar. Through a preference or some other method, users would be able to change this set of commands to an alternate predefined set.

Resizing using direct manipulation

    The user should be able to resize the main toolbar which contains icons and text stacked over one another. The resize cursor appears when the user mouses over the edge of this toolbar.
    Resizing the main toolbar

    As the user drags up, the toolbar "snaps" into the next position. If the toolbar originally showed Pictures and Text, it will now show Pictures Only, and so on. Likewise, if the user drags down, the toolbar "snaps" into the next largest position.

    The "Show Toolbar As" preference is updated accordingly.

    If the animation is on this tab, and resizing forces a smaller area, the animation must shrink to fit the new area. Note that the Location toolbar and the user-defined toolbar are the same size and take on the small animation when one of them is the top toolbar. If the command toolbar is at the top, then the large animation is used only when the toolbar is in Pictures and Text mode.
