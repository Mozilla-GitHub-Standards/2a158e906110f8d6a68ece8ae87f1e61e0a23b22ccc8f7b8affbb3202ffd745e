UCF: the URL Connection Factory

There is one URL Connection Factory type per protocol, and they all implement this interface. This interface has two methods: one which returns the name of the protocol, in the form used in URLs, and one which creates URL Connection types.

Definition

This interface is defined in java, and is compiled with JMC. The C syntax is included in each method description, and is summarised below. 
package netscape.net;

import netscape.url.URL;

public abstract interface UCF {
  public abstract String getProtocolName();
  public abstract UC createUC(URL url, NC nc) throws Exception;
}
UCF.java

Interface Methods

  • There is no creation method. This is an interface. Each per-protocol object, however, has a creation method; see the actual protocol implementations for descriptions.
  • getProtocolName

    • This method returns the name of the protocol, in the form used in URLs. E.g., "http," "ftp," "nntp," etc. Without the colon. When these factories are registered in the URL Connection Factory Registry, the registry actually matches these names against URLs.

    The C signature of this method is:

    const char *UCF_getProtocolName(UCF *self, JMCException **e);

    Note that although the java method signature does not specify that an exception may be thrown, due to the existance of java errors, an error may be reflected in this method's final argument.

  • createUC

    • This method actually creates a protocol-specific object that implements the URL Connection interface UC. The arguments are:
    • URL *url: The URL describing the web object and operation, and
    • NC *nc: An object supporting the Net Context NC interface, used to provide resources to the network operations.
    The C signature of this method is:
    UC *UCF_createUC(UCF *self, URL *url, NC *nc, JMCException **e);

Inherited Methods

Although this interface does not inherit anything, note that all objects implementing this interface will inherit the usual JMC base methods.
