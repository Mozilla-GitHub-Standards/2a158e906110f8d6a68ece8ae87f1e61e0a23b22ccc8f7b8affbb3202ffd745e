Legends

The status label indicates fit-of-use and progress of the document. More labels may be created as needed.

Status labels
Current The document contains complete and up-to-date information on recent source releases and the current development process.
Stable The document is stable and applies to past milestone source releases or technical specifications.
Outdated The document is outdated but may contain information relevant to recent source releases and the current development process.
In-progress The document has not been completed but either someone is actively working on it or the document is updated frequently.
Incomplete The document is incomplete but no one is working on it. The document may be outdated.
Draft The document is a draft version and we don't know if anyone is updating it or if it is up-to-date. The document may or may not be complete.
Brief This is a short note which may or may not be up-to-date.
Obsolete The document has no useful information for current development and is kept mainly for historical reason.
Null We haven't seen anything material of this document.

The status colour code indicates if help is needed on working on the document but not the urgency for such work. Colour code are for reference only.

Status colour code
green The document has all the attention it should have. Normally current and stable documents are green.
orange The document needs work, and either its bugs are ASSIGNED or someone is working on it.
red The document needs work, but nobody is doing it.
