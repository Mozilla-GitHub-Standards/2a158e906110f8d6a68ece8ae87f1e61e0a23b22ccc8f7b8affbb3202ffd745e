Mozilla Documentation Project homepage

Documentation tools and templates

There are various resources contributors can use for preparing and submitting documentation to the Mozilla Documentation Project, as described here.

Guidelines for contributors

See the Contribute Documentation article for more details about the logistics of contributing to the documentation project. Also, be sure to review the mozilla.org Documentation Style Guide.

Bugzilla documentation area

To track documentation issues and to provide feedback on a particular document, a Documentation project area has been created at Bugzilla.

Documentation newsgroup

The documentation newsgroup is an excellent resource for posting questions or obtaining information. Visit the newsgroup at netscape.public.mozilla.documentation.

Format for contributions

The current preferred format for contributions is HTML 4.01 strict.

Tools for authoring documentation

We don't provide or endorse any particular tool for producing documentation. Use whatever works for you for producing HTML documents, as long as documents validate as HTML 4.01 strict. Be aware, however, that WYSIWIG HTML authoring programs can insert unwanted line spaces into your document. Consider using BBEdit on the Mac, or KompoZer 0.77 (or higher) on Windows, for your authoring environment. Some lengthier documents for the documentation project are being authored in FrameMaker and converted to HTML by using WebWorks Publisher.

Templates for contributors

There are several different HTML templates you can use to author contributions to the Mozilla Documentation Project:

The Mozilla Document Template can be used for general documentation. Note that there is a specific template for reference documentation, described next.

The Mozilla Reference Documentation Template can be used for API reference information.

The Mozilla Table of Contents Template can be used for a frameless, multipaged document that requires a table of contents or for a document with a single page. In general, if your document is greater than 10 printed pages, you should use a table of contents. However, if you expect that the document will be read primarily online, and the document contains multiple sections, than consider using a table of contents even if the document is less than 10 pages.

Indexing tool

The Documentation Indexing Tool is a Perl script that "can make your indexing a lot easier when the time comes."
