Documentation Indexing Tool

Script Source
Script

It's not very sophisticated, but the Perl script included with these templates can make your indexing a lot easier when the time comes. The script needs to be configured a little bit for your system and the location of your content, but once this is done, you can run it as often as you want to pick up new entries. Each time the script runs, it generates a new index page. Note that the script requires that you have Perl installed to run at all.

The script scans every html file in the directory you specify and compiles a list of special index tags in those files. The index tags are anchors to which links in the index page will point; they are alphabetized and pretty-printed in a file called (unless you name it something otherwise) ref_index.html, which will sit in the content directory with the other files. The indexing is also recursive, so if you have content subdirectories, the script will search these out as well.

To use the script, edit the target directory and start it from the command line as follows: 

perl index.pl [new name of generated index]

If you do not supply an argument, the script creates the aforementioned ref_index.html. It does not edit any of the source files, but it looks in every one for anchor tags that you have supplied. If you do not create any index entries in your content files, the index will be empty.

The format for the index entries in the content pages is as follows: 

<a name="index stringIDX"></a>

The IDX must appear at the end of the string and within the quotes or the script will not find the entry. When the script finds a target formatted like this, it creates an index entry with the index string and strips off the IDX itself. You can test the script by creating a single index target in one of the document, running the script (after you have specified the target directory), and viewing the results in the ref_index.html page.

The following example shows a single index target in an HTML file and the resulting index page, small though it is: 

<html>
<head>
<title>Sample Content Page</title>
</head>
<h1>Installing Your ExCo SDK</h1>
<p>The ExCo SDK comes with a special 
<a name="Installation ToolIDX"></a> 
installation tool that makes it easy to ....

I
Installation Tool

The indexing script is a strictly one target, one item tool. You can make as many separate targets in a single area as you want but not, I think, more than one target per line. And each target must be unique or the later or the two will replace the earlier.
