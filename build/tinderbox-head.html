Checkin requirements

  • Mozilla contributors who are not Netscape employees: submit patches to brendan@mozilla.org or waterson@mozilla.org for checkin approval (often available on #mozilla).
  • Netscape employees: dogfood+ and nsbeta3+ bug fixes only. Manager + marketing counterpart approval needed needed to nsbeta3+ a bug.
  • Do not checkin on RED.
  • Run pre-checkin tests, if you have time run the smoketests.
  • Get permission from the module owner.
  • Checkin comment should include a reviewer (e.g. r=email), a clear explanation of the fix, and a bug number if there is one.
  • Add new files to Mac/Win/Unix packages!

Help

  • See Bonsai for checkin details.
  • Ports has build status on other platforms.
  • Follow the rules and hints for landing changes.
  • If problems develop, send email to the hook and contact the sheriff.
  • Add search engines: mozilla.org & Bugzilla & LXR & dmoz.org
  • Add tinderbox to your sidebar.
  • If the tree is still closed, see the smoketest blockers.
