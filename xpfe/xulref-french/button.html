value
Description
value est un attribut exigé employé pour fournir une étiquette au bouton.
Syntaxe
<button value="string" />
Exemple
<button id="pushme" class="push" value="Push Me" />
Notes
Il est légal de ne pas fournir de valeur pour un bouton, mais le bouton peut être invisible s'il n'y a pas valeur pour contraindre la taille réelle du bouton ou aucune information concrète qui indique ses position et taille.

 

 
class
Description
class est un attribut qui indique le modèle et le cadrage d'un bouton.
Syntaxe
<button class="classe dans global/button.css ou autre fichier css" />
Exemple
<button id="pushme" class="left" value="Push Me" />
Notes
Les classes les plus communes du skin global sont:
  • button.small
  • button.left
  • button.right
  • button.top
  • button.bottom

 

 
onclick
Description
onclick est un attribut auditeur d'événement utilisé pour ajouter une fonctionnalité aux boutons XUL. 
Syntaxe
<button onclick="code javascript ou appel à une fonction javascript"
Exemple
<button value="Print" id="print" onclick="alert('Printing Document'); Print();" />
Notes
L'auditeur d'événement onclick est une manière d'appeler les commandes des boutons. Notez que oncommand est la manière préférée d'attacher un dérouleur d'événement à un bouton. onclick est désapprouvé, bien qu'encore disponible.

L'événement du clique est actionné toutes les fois que l'utilisateur clique le bouton. Vous pouvez mettre du JavaScript en ligne avec l'auditeur d'événement, vous pouvez appeler des fonctions JavaScript définies dans la section <head> du fichier XUL dans un élément <html:script>...</html:script>, ou vous pouvez appeler des fonctions JavaScript définies dans un fichier source séparé qui est chargé avec la construction <html:script language="javascript" src="source_file_name.js"/>

 
 
 
oncommand
Description
oncommand est un attribut auditeur d'événement utilisé pour ajouter une fonctionnalité aux boutons XUL. 
Syntaxe
<button oncommand="code javascript ou appel à une fonction javascript"
Exemple
<button value="Print" id="print" oncommand="alert('Printing Document'); Print();" />
Notes
L'auditeur d'événement oncommand est une manière commune d'appeler les commandes des boutons. Elle substitue onclick comme principale manière d'attacher un dérouleur d'événement à un bouton de titre.

Vous pouvez mettre le JavaScript en ligne avec l'auditeur d'événement, vous pouvez appeler des fonctions JavaScript définies dans la section <head> du fichier XUL dans un élément <html:script>...</html:script>, vous pouvez appeler des fonctions JavaScript définies dans un fichier source séparé qui est chargé avec la construction <html:script language="javascript" src="source_file_name.js"/>.


 
tooltiptext
Description
tooltiptext est un attribut facultatif employé pour indiquer le texte tooltip pour le bouton.
Syntaxe
<button tooltiptext="string" />
Exemple
<button id="send_mail" class="push" tooltiptext="Send Mail (Ctrl + M)" />
Notes
Si vous voulez afficher du texte plus long ou structuré dans la fenêtre de tooltip, vous devez utiliser l'attribut tooltip à la place.

Syntaxe
<button default="true | false" />
Exemple
<button id="quit" default="true" />
Notes
Aucune.

toggled
Description
toggled est un attribut facultatif employé pour indiquer l'état  basculé d'un bouton.
Syntaxe
<button toggled="0 | 1 | 2" />
Exemple
<button id="local" class="push" toggled="1" />
Notes
0 signifie qu'il n'est pas basculé, 1 signifie qu'il l'est et 2 signifie que les deux sont possibles.

 

 
src
Description
src est un attribut facultatif pour le chargement d'image du bouton.
Syntaxe
<button src="URL" />
Exemple
<button id="security" src="images\lock_and_key.gif" />
Notes
Aucune.

 

 
crop
Description
crop est un attribut facultatif pour spécifier la partie texte d'un bouton. Le cas échéant, il sera coupé s'il est trop grand.
Syntaxe
<button id="string" value="Button Name" crop="left | right | none | center" >
Exemple
<button id="string" value="Très long nom de bouton" crop="none" >
Notes
Aucune.
