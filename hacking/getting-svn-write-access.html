Getting SVN Write Access to www.stage.mozilla.com For Localization of In-Product Web Pages

This document describes steps required to obtain SVN write access to www.stage.mozilla.com for the localization of in-product web pages for Firefox 2. This document applies to all potential contributors.

Logistics

For those in a hurry, here's a list of the steps that need to happen to get SVN Commit Access to www.stage.mozilla.com. Explanations are in the "How & Why" section below.

  1. Download the archive of the files to translate from the attachments list at Bug 354498.
  2. After extracting the files, fill in the form titled contributors.txt that is contained in the archive with your contact information.
  3. Email the completed form to Pascal Chevrel, localization manager for the in-product web pages project. Alternatively, file a Mozilla IT bug requesting SVN access to www.stage.mozilla.com and attach the completed form to the bug. Note: if no form is attached, the bug will be closed, so please don't forget that step.
  4. An appropriate member of Mozilla staff will update the bug to say whether s/he has approves of the request.
  5. Mozilla IT will double-check that the needed info is recorded and, if so, create an SVN account for access to www.stage.mozilla.com.
  6. Mozilla IT will update the bug to note the account has been created, and close the bug.

How and Why

SVN write access to www.stage.mozilla.com is currently limited to localizers working on translations of the new in-product web pages that are being introduced with Firefox 2. Read more about the in-product pages on the Mozilla Wiki.

We're following a different process for providing authorization to localizers to access this staging server because the content being managed and committed by localizers for their translations is not going to be licensed under the tri-license, and does not have the same prior art or ownership restrictions since it is not source code.

This entire process applies to everyone who wants SVN write-access to www.stage.mozilla.com for the localization of in-product web pages.

Problems With Your Account

  • If you find that someone is using your account, notify us of the problem by filing a server-ops bug. This will get routed automatically to Mozilla IT.
