ECMAScript Edition 4 Drafts

Monday, June 30, 2003

ECMAScript Edition 4 working drafts maintained for the ECMA TC39 TG1 working group are listed below.

Date Links Revisions
Jun 30, 2003 Word PDF
  • Implemented most of the library algorithms.
  • Fixed problems with the core algorithms and reorganized them slightly to help in writing the library algorithms.
  • Added support for supplementary Unicode characters.
  • Implemented simple prototypes on most objects, including class objects themselves, for compatibility with Edition 3.
  • Readded package and import directives.
May 2, 2003 PDF
  • Simplified some of the semantic object data structures. Merged uninitialised with none. Defined class initialization. Defined getters and setters. Implemented instanceof.
  • Defined error classes and replaced error tags with these classes. Revised which error gets reported in many cases.
  • Renamed class semantic variables from names such as objectClass to Object.
  • Removed multiple constructors, the constructor attribute, and the invoke superconstructor case.
  • The return statement cannot return a value from a constructor or a setter. A getter must return a value; it cannot fall off the end of the function.
  • Added the ... function call syntax for passing an array of arguments to a function.
  • Removed package, import, and export directives.
  • Implemented the remaining core language algorithms.
Mar 24, 2003 PDF
  • Implemented the remaining statements (with, catch, for, and for-in).
  • Added support for the enumeration protocol; simplified spec by removing some enumerability choices.
  • Implemented instance methods on classes
  • Implemented function call and object creation logic.
  • Finished and reorganized the property read, write, and delete code and indirected it through six customizable internal methods.
  • Reorganized instance layout to fix design flaws with distinguishing compile-time constants from run-time constants.
  • Restricted instance variable initializers to compile-time constant expressions.
  • for-in statements may not have useless initializers.
  • Removed support for named arguments.
  • Implemented array initializers and updated object initializers.
  • Merged packages with global objects.
  • Merged the concepts of inaccessible and uninitialized variables.
  • Allowed dynamic properties with namespaces.
  • Simplified the override logic by introducing the restriction that an instance variable cannot override a getter or a setter.
  • Numerous smaller changes and additions.
Jan 13, 2003 PDF
  • Major changes and additions throughout the document.
Jun 11, 2002 PDF
  • Additions to semantics of member lookup, frames, statements, and definitions.
Apr 23, 2002 PDF
  • Major updates and additions to semantics of statements and expression evaluation.
Nov 28, 2001

New draft in PDF with no change bars

Raw algorithm change document in PDF
  • Changed validation and evaluation code to use contexts, static environments, and dynamic environments.
  • Miscellaneous minor revisions to the evaluation algorithms and data structure.
  • Added semantics for attributes.
  • Added semantics for most statements.
Oct 26, 2001 PDF
  • Made bulk changes in the semantics to address issues brought up at the Oct 1 meeting.
  • Implemented property reading and writing.
  • Added indices and table of contents.
Sep 26, 2001 PDF
  • Added early drafts of chapters 11, 12, and 19. Haven't written many explanations in these chapters yet. These are bulk changes, so they don't have change bars.
  • Miscellaneous changes throughout the document marked with change bars.
Aug 22, 2001 Word PDF
  • Rewrote sections 5.1 through 5.12.1 and added chapters 9 and 10. These are bulk changes, so they don't have change bars.
  • Removed 3_in lexer syntax.
  • Updated list of keywords.
  • Removed line numbers from algorithms and made other small style sheet changes.

  • Added outline of all chapters.
