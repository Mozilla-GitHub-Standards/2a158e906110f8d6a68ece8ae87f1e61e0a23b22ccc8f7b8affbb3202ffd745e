Mozilla Security

A web browsing and e-mail experience should not be unsafe, and your data should never be compromised. Mozilla is not prone to the same kinds of security vulnerabilities that other browsers are prone to, many of which rely on tight integeration of the operating system, less secure default settings, and easy methods for remote code download and execution. 

The core of the mozilla architechture relies on sandboxing from the OS, protective default settings, and no activeX script support to avoid many potencial security vulerabilities.  Mozilla includes many features which help to protect security and privacy vulerabilities and annoyances.

The Mozilla development process assists in reducing risks from security problems.  Aspects of that development process include:

  • Audits of the entire source code baseline are conducted to identify problems
  • Extensive peer review reduces the chances of introducing new vulnerabilities
  • Academics and world renowed security experts study the source code and programs independently and help the project to identify and reduce the chances of vulerabilities
  • Security@mozilla.org mail alias set up to receive incoming reports of potencial vulerabilities and allow our security team to evaluate and fix potencial problems.
  • Any discovered vulerabilities are fixed quickly and made available for deployment.
  • Openess and transperancy often lead to trust, and we believe help Mozilla to provide the most secure browser, e-mail, and internet application platform.
    •

Independent evaluations consistantly show Mozilla is the most secure browser.
