Mozilla Software Integrity Policy

**DRAFT** version 0.1

Policy for software hosted by Mozilla Addons

1. All hosted software must be in an acceptable source format. It does not have to be open source, but code must be licensed such that it can be viewed and read by both mozilla.org staff and potential end-users.

2. A Privacy Policy must be displayed and agreed to by the end-user during installation if any hosted software collects (e.g. keystroke logging, network monitoring, etc.) or disseminates user data or if network connections beyond user initiated normal browsing are made.

3. All hosted software shall comply with the Mozilla Software Publishing Agreement (Ed: legal stuff, under development).

4. All hosted software must be of equivalent quality and stability to mozilla.org code.

5. All software updates must be handled through addons.mozilla.org. External update URLs are not acceptable.

6. All hosted software must not interfere with the normal operation of any third-party services in such a way as to be considered malicious or damaging. Additionally, hosted software that interacts with specific services must not violate the terms of use of those services.

7. All hosted software must function as described, without additional behaviour and/or side effects, intentional or not. Anything that changes the core behavior of the host application must be clearly documented to the end user.

8. We may refuse to host software if upon review it is deemed to be unstable, poorly written, or if it performs a function for which there is no percieved need.

Above all else, software hosted by mozilla.org must not do anything that compromises the integrity or security of the end-user.
