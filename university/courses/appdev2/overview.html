Course Overview
MU 150: Extending Your Mozilla Application

  • Prerequisites
  • Conceptual Overview
  • Lessons

This document describes a series of lessons for extending Mozilla applications with XPInstall, RDF, DTDs, overlays, and other open source technologies. It presumes that you have already completed Mozilla University's Application Development 101 (MU 101) or are competent with the technologies described in that series of lessons. Most or all of the technologies in this course extend or relate directly to applications that have been created with XUL and JavaScript.

This course does not include lessons on XPCOM or component development, which are part of a separate, later course in Mozilla application development (MU 250 Component Development). The prerequisites for MU 150 are discussed further in the section Prerequisites.

Course Goals

Students who take this course or follow along with the series of lessons here on their own ought to be able to:

  • Easily create basic Mozilla applications
  • Package applications
    • Archive packages in JAR files using the proper structure
    • Understand the XPI file format and its behavior with the browser
    • Manipulate the registry using installed-chrome.txt
    • Create separate packages for the content, locale, and skin of an application
    • Use XPInstall to register new chrome with the chrome registry
    • Create installations scripts (trigger and install.js) for applications
  • Localize/internationalize Mozilla applications
    • Use DTD entities in XUL
    • Use string bundles in scripts
    • Packaging locales
    • Localization APIs
  • Use dynamic data in your application
    • Use overlays to put new content and behavior in the browser

    Prerequisites

    To follow along effectively in this course, you should be able to use XUL to create basic client applications. This means that you are already familiar enough with the chrome registry system to create a new package that Mozilla recognizes and can load as chrome.

    Conceptual Overview

    As you will already have learned, the tools in the Mozilla cross-platform front end (XPFE) allow you to rapidly prototype, build, and extend cross-platform applications. Mozilla application developers typically use XUL, CSS, and JavaScript to define the structure, presentation, and behavior of their applications, respectively. These are the basic tools described in Application Development 101 (MU 101)

    The first extension to this set of technologies addressed in MU 150 is the Mozilla installation technologies, since these are required in applications that are registered with Mozilla and accessed as chrome. As applications become more complex they are divided up into different modules that can be maintained separately. A theme, for example, is a set of CSS and image files that can be dynamically swapped in a Mozilla application because the presentation layer has been modularized in this way. The underlying structure of the application is untouched.
