Navigator Team Progress for the week ending 10/12/ 2001

Highlights

  • Samir Gehani implemented persistent panels in sidebar.
  • The Navigator team resolved 230 Bugzilla bugs in the last week, fixing72 of these.  
  • The  MachV engineering plan for Navigator is now split between  publicly-viewable items posted on Mozilla.org, and a few Netscape confidential items posted only internally. We also have a draft schedule , though we are in the process of doing some serious resource levelling to get the workloads right.  Still in the process of creating  tracking bugs for all the high-level MachV items, please point out any existing bugs you may know of that fit, so we can avoid creating duplicates.  If you have any interest in our MachV work, please review these documents  to ensure that our plans will meet your needs, and let me know of anythying you'd like clarified or changed.

Lowlights

  • Too many bugs targetted for 0.9.6 ( 226 in bugzilla  ).  We need to get the number of non-feature work bugs down to about 90, in order to have time for planned performance and MachV feature work, let alone incoming bugs.
  • Still swamped by untargetted bugs, 626 in Bugzilla .  Several engineers have been triaging fast enough to catch and keep up though, with help from QA and Mozillians.

          Paul Chen (pchen):

  - Thanks to GlowCode, got some profiles of back performance for going 
back from 
www.yahoo.com  to 
www.netscape.com . According to GlowCode,  we're spending 5-7x times
longer building up the content model than we  are displaying it. For example,
on my P3 500 system at work, The main  thread spends 5478ms in PL_HandleEvent.
Approximately 98.5% of that time  is spent building up the content model.
On top of that time,  nsWindow::ProcessMessage (called by nsWindow::WindowProc)
takes 934ms,  78% of that time is spent in nsWindow::OnPaint and 18.6% is
spent in  ChildWindow::DispatchMouseEvent. Need to look at other pages, especially
 those without or with minimal javascript to see if there are any  significant
profile differences. 

 - Working on bugzilla 80208, reduced popup listeners by 23 in the nav  window 
but then realized going home Friday that I screwed up tooltips in  all status/component 
bars outside of nav window. Since this is kinda  cleanup work before Hyatt 
lands 93839, I'll back out the  status/component bar changes so that I don't 
touch so many files (i.e.  mailnews window xul files).
