Navigator Team Progress for the week ending 11/09/ 2001

Highlights

  • Navigator team is focussing on performance, with some good results:
    • Steve Morse completed the implementation of converting cookie/image/password managers to use outliner, with order of magnitude speed improvements.
    • Paul Chen improved time to open new windows that have bookmark folders on personal toolbar, saving nearly one-half second per 100 bookmarks.
    • Bill Law created a chromeless browser window and related tools that establish a baseline for performance and help us improve it.
    • Brian Ryner sped up nsLookAndFeel::GetColor.

  • The Navigator team resolved  210 bugs last week, 208 in Bugzilla, fixing 56 of these.

Lowlights

  • Still having a bit of a hard time keeping up with our untargetted bugs, 290 in Bugzilla.  
  • Still have bugs targetted for mozilla1.0,   52  in Bugzilla

                Bill Law (law):

   Got up to speed on "Cost of Chrome" performance work; coded up a  "chrome-less" 
browser window and related tools; see them at  mozilla/tools/performance/startup 
(you have to "cvs checkout" this  project explicitly).  This gives us 
a  "MOZ_TIMELINE" (tool that logs cost of various  things) baseline for the 
cost of starting up Mozilla independent of the  Navigator chrome (or as independent 
as is currently feasible).  On my  machine, the time to start up (and 
close) this minimal Navigator window  is 3.297 seconds (versus 7.312 seconds 
for the full Navigator window). 

    

                Steve Morse (morse):

   

- Cookie/Image Manager is slow
Completed the implementation of converting cookie/image/password
managers to use outliner.  Also cleaned up a lot of coding
inefficiencies in these modules.  As a consequence of both of these
changes, the speed improvements were significant.

Unfortunately it involved almost a complete rewrite of the entire
modules.  Consequently, much legacy code which has nothing to do with
performance improvements had to be redone as well because it would never
get by today's review requirements.  This made the rewrite take
considerably longer than would have been necessary for simply improving
the performance.

- Full Screen Mode
Finally got implementation approved by reviewers and patch will be
checked in as soon as the tree reopens for post 0.9.7 checkins.  The bad
news is that the nav buttons in full-screen mode are still large images (original patch converted them to text-only but reviewers didn't like the way that was done).  So the buttons will remain as images until an infrastructure change is made that allows text-only buttons in a manner that is acceptable to
reviewers.  (See bug 68136.)

- Global XUL cleanup
Unscheduled work due to a lot of fallout from the xul cleanup that
occurred throughout the tree last week-end.  Found and fixed many
problems affecting password manager, cookie manager, and form manager.
