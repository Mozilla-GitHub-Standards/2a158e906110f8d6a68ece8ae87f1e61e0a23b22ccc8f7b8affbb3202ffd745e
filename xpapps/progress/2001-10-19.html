Navigator Team Progress for the week ending 10/19/ 2001

Highlights

  • Brian Ryner finished his tabbing rewrite, which fixes several major navigation bugs.
  • Joe Hewitt has been plowing through his XUL 1.0 work.
  • Dave Hyatt improved page load times by 10% on win32 by mucking with the priority of messages.  Dave also added prefs and background loading for tabbed browsing.
  • Blake Ross is making rapid progress converting Bookmarks to outliner, details in his report below.
  • Samir added a close button to the sidebar, one of our most requested usability improvements.
  • The Navigator team resolved 172 Bugzilla bugs in the last week, fixing 22 of these.  No current eMojo bugs on our radar, nominated or otherwise.
  • The  MachV engineering planning for Navigator is winding down, as we ramp up its development.  We've adjusted the resource leveling, and are now in the process of creating  tracking bugs for all the tasks.  

Lowlights

  • We only have Matt Fisher on the Netscape payroll for one more week.  :`-(
  • Too many bugs targetted for 0.9.6 ( 171  in bugzilla ).  We need to get the number of non-feature work bugs down to only those we are committing to, in order to have time for planned performance and MachV feature work, let alone incoming bugs.
  • Still swamped by untargetted bugs, 494 in Bugzilla .  Need to reduce these to zarro, pronto.

         Ben Goodger (ben):

 

- checked in accumulated fixes for 80150, 75825, 87864, 94239, 74036 (Becki 
Sanford), and 91073 (Henrik Gemal) (some patches from last week, some 
contributed)
- made significant headway on "Save with Images"/"Save as Text", 
specifically:
 (1) Added ability to set/get which filter is selected in Windows file picker
 (2) Implemented UI for "Save with Objects" using embedding APIs. 
 (3) Implemented HTML->Text converter component which wraps nsIHTMLToTextSink 
for conversion of strings (and, sometime in the future, streams) from HTML to 
text. 
 (4) Modified call sites in browser (file menu, shift+click, context menus) 
to use new "Save" function
 (5) Gave save function the ability to be able to differentiate between 
document and invocation types, so that the filter set offered can differ to 
provide meaningful options. Connected this to the MIME datasource so that the 
filter list can show OS-like file types in the filter box (e.g. choose "Save 
Image As.." on a .jpg image and you'll see "JPEG Image" as the default 
filter. 
  This all more or less works, bar a couple of bugs. 
- Handed off bookmarks outliner to blake, gave him a brief introduction to 
the path I was taking, and showed him my existing (slightly out of date) 
documents so he could plunder them
