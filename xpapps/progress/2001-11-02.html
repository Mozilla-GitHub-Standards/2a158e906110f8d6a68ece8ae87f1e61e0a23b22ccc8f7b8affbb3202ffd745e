Navigator Team Progress for the week ending 11/02/ 2001

Highlights

  • Navigator team is focussing on performance, with some good results:
    • Brian Ryner improved page load times 2-3% by eliminating some unnecessary painting.  
    • Brian also made the filepicker widget 8X faster.
    • Steve Morse made converted Cookie Manager to user outliner, allowing cookies to load 10X faster, and making several lengthy operations instantaneous.
    • Paul Chen added John Morrison's XUL open window tests to Tinderbox, with help from Chris McAfee.  We can now spot performance regressions there within minutes.
    • Paul also deferred bookmark loading at startup until after a browser window opens, cutting startup time by up to a second, or more,  although YMMV.
  • The Navigator team resolved 172 Bugzilla bugs in the last week, fixing 25 of these.  

Lowlights

  • Too many bugs targetted for 0.9.6 ( 57 in bugzilla  ).  
  • Still digging out from under our untargetted bugs, 210 in Bugzilla.  
  • Still have bugs targetted for mozilla1.0,   108 in Bugzilla.

Paul Chen (pchen):
- Added ability to run jrgm's xul open window test to tinderbox. As it's currently configured, it opens and closes ten browser windows with a minimal html file to time the open. It averages the time of the 2nd through 10th window open times and reports that time. If you want to take a look, it's currently staged on MozillaTest tbox page http://tinderbox.mozilla.org/showbuilds.cgi?tree=MozillaTest. As of this  moment, it's running on coffee and lespaul. Just look for "Txul:xxxxms". Thanks to mcafee for helping me get it into tinderbox.

- Fixed bugzilla 68239, updated string handling code in nsBookmarksService.cpp, thanks to jag for his string-fu knowledge.

- Fixed bugzilla 81762, minor replacing of strlen() calls with null and non-empty string checks.

- Have patches for delay loading of bookmarks until browser window opens. Was going to get it in today, but I think I'll gauge more people's thoughts on whether this is really worth taking (though almost everyone around the office says go for it -- as long as the time to have the personal toolbar items show up doesn't take a long time).

            Steve Morse (morse):

 

Did prototyping and determined that a conversion to outliner will result
in nearly a 10:1 speed improvement for a cookie list of 200 cookies.

Did actual implementation and observed the following results for a
cookie list of 200 cookies, 200 cookie permissions, and 200 image
permissions.

   load time for cookie manager dialog:
      before: 23.4 seconds
      after:   2.7 seconds

   clicking on remove-all cookies button
      before: 5 seconds
      after:  instantaneous

   sorting cookies:
      before: 2 seconds
      after:  instantaneous

   selecting all cookies and clicking on remove button
      before:  37 seconds
      after:   instantaneous

   clicking on remove-all sites button
      before:  9 seconds
      after:   instantaneous
